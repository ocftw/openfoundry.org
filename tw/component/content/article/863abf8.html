澳洲維多利亞省政府部門採用開放源碼

建立日期 2007-03-06 08:00      最近更新在 2007-07-05 19:36 作者是 謝良奇
澳洲維多利亞省 (Victorian) 政府部門正逐漸與當地開放源碼開發服務商合作，在客製化應用軟體上，借重社群的力量以提高軟體標準，如營繕部 (Department of Infrastructure) 與環境永續部 (Department of Sustainability and Environment) 都已經是當地開放源碼軟體開發服務公司的客戶。 墨爾本 (Melbourne) 的軟體開發服務商 Groupware Consulting 為維多利亞省政府單位開發的客製軟體，採用開放源碼元件像 Spring 框架與 MySQL 資料庫。該公司設計與交付經理 Domenic Brasacchio 表示，開放源碼所帶來的優點是他們將重心放在開放源碼上的主因。

Brasacchio 認為維多利亞省已受惠於社群的貢獻及這些軟體的專業級水準，開放源碼符合企業軟體的各項標準如延展性、可靠性與可移植性。Brasacchio 認為，Sun 將 Java 開放源碼的決定顯示了軟體服務的走向。當更多企業與政府了解到開放源碼的優點，開放源碼與軟體服務化 (software as a service，SaaS) 的發展就會更加快速。

Brasacchio 表示，企業的軟體專案假如有 80% 採用既有元件，20% 使用客製開發，往往那 20% 的部份會使專案成本增加近 3 倍之多。

他舉 Spring 框架為例說明開放源碼的優點，他說，Spring 由於提供了各式核心元件，可以達成快速軟體開發的目標。

Groupware 同時也透過釋出 Spring Java 框架的延伸元件，貢獻開放源碼社群。Spring Layout、Spring MVC 與 tag 延伸元件都是經營在 Sourceforge.net 上的開放源碼專案。Groupware 表示，該公司的開發人員幾個月來都在發展 Spring的 tag 程式庫與框架延伸元件，可以幫助資料中心 Web 應用軟體的快速開發。

該公司也正準備釋出自行開發的事項追蹤與專案管理軟體 iTrack 為開放源碼軟體。Brasacchio 表示 iTrack 還在 alpha 階段，應該會 在下一季完成。有不少維多利亞省的政府部門已經在使用 iTrack，這套 Groupware 原開發用以讓客戶管理、追蹤與記錄專案事項的軟體，可協助專案規劃、成為用戶與服務供應者的溝通工具並管理應用軟體測試及需求變更程序。
