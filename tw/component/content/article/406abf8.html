Oracle 收購開放源碼資料庫商 Sleepycat Software

建立日期 2006-02-16 08:00      最近更新在 2007-03-08 07:58 作者是 謝良奇
Oracle 不久前收購了開放源碼資料庫廠商 Sleepycat Software，驗證了日前該公司將收購多家開放源碼公司的傳聞，除了增強 Oracle 在嵌入式資料庫市場的實力，也有更進一步挖競爭對手 MySQL 牆角的意味。 這次的收購案擴展了 Oracle 的嵌入式資料庫產品線，包括行動裝置使用的 Oracle Lite 與該公司去年收購的高效能記憶體資料庫軟體 TimesTen。至於收購案本身的價錢則並未揭露。Oracle 表示，本收購象徵用戶可以用低廉成本使用快速的開放源碼資料庫，同時又享有企業級的支援。

Sleepycat 的 Berkeley DB 為多種知名開放源碼產品中常用的嵌入式資料庫，包括 Linux、BSD 作業系統、Apache 伺服器、OpenLDAP 與 OpenOffice。Sleepycat 執行長 Mike Olsen 表示，他們很高興與全球最大企業軟體公司攜手合作。Oracle 的資源將可提升該公司對用與開放源碼社群的服務品質。

數週來已有許多傳言認為 Oracle 在設法收購多家開放源碼公司，BusinessWeek 報導中指出的收購對象包括了 Sleepycat、JBoss、Zend。Sleepycat 並不是 Oracle 收購的第一家開放源碼公司。去年該公司才收購了開發 MySQL AB 資料庫核心 InnoDB 的 Innobase。這項收購案被解讀為是正對著 Oracle 競爭對手 MySQL 而來。

話雖如此，開放源碼大將 Bruce Perens 倒是有自己的一番見解。Oracle 連番收購了兩家 MySQL 交易資料庫後端的開發廠商，照理說對 MySQL會造成不小的傷害。但是 MySQL 仍然可以繼續以開放源碼授權方式採用這兩家公司的產品。因此 Oracle 藉此打擊 MySQL 的希望也將隨之落空。

Bruce Perens 認為完全買下一個開放源碼專案是不可能的，GPL 等開放源碼授權的設計就是讓其它參與者能不受他人的阻礙。Larry Ellison（Oracle 執行長）可以買下一間公司並影響一個開放源碼專案，一旦他想要獲取完全的控制權時，開放源碼開發者就會搬到別處另起爐灶。

Sleepycat 執行長 Mike Olson 在開放源碼商業大會（Open Source Business Conference）上面對與會者提到收購開放源碼公司時企業與開發者間的不確定感時，他說，如果你希望培育一個社群，就要留心維護誠信與熱情。絕不能 忽視。
