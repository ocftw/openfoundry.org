紅帽釋出 RHEL 5.5 新增最新處理器支援

建立日期 2010-04-06 08:00      最近更新在 2010-06-07 22:39 作者是 謝良奇/編譯

開放源碼解決方案供應商紅帽 (Red Hat) 發表了 Red Hat Enterprise Linux (RHEL) 5 平台的第 5 次更新，Red Hat Enterprise Linux 5.5。在新版本中，AMD、Intel、IBM 最新處理器的支援，被向後移植 (back-ported) 到了 RHEL 5 堆疊的 Linux 2.6.18 核心上。

此次更新增加了設計用來跨越實體、虛擬、雲端部署的功能，伴隨著對新硬體平台的支援，提供虛擬化和相互運作性的功能加強。和所有紅帽的更新相同，相容並通過 RHEL 5 平台認證的應用軟體，將可順利運作於 RHEL 5.5 上。RHEL 5.5 將立即透過 Red Hat Network 提供給訂戶使用。

除了整體平台增強與臭蟲修復外，此次更新針對來自紅帽合作夥伴的新平台提供了支援，這些夥伴包括 AMD、Dell、HP、IBM、Intel。新支援平台包括 Intel Nehalem EX (Xeon 7500 系列)、AMD Opteron 6000 系列 (先前為 "Magny Cours")、IBM Power 7。紅帽平台工程副總裁 Tm Burke 表示，RHEL 5.5 的核心經過改善，納入了新近 Linux 核心的功能，稱呼它為 Linux 2.6.18 已經不太恰當了。

AMD 軟體解決方案行銷經理 Margaret Lewis 表示，RHEL 和 AMD 平台共同提供用戶資料中心可靠性與彈性。RHEL 5.5 的最新技術加強結合新的 8 與 12 核心 AMD Opteron 6000 系列平台，能夠提供用戶更強大效能並維持低電力消耗。

由於基於 Opteron 6100 系列、Xeon 7500、Power7 處理器的機器，都採用了非一致性記憶體存取 (non-uniform memory access，NUMA) 跨越多重處理器共享記憶體，而且在處理器中也擁有多重核心與快取，RHEL 5.5 之中做了許多工作讓作業系統得知系統拓撲，以便在記憶體配置與工作排程中，儘可能讓指令流所在位置接近所需資料。而且核心也經過修改，以較少的核心完成工作，允許伺服器藉由關閉其他核心以節省電力消耗。

更新後的 RHEL 可利用現今許多 x64 和 Power 處理器中的虛擬 I/O 硬體功能，提供 I/O 最佳化，減少虛擬化環境中的 I/O 開銷。Burke 指出，藉由這些針對 I/O 虛擬化的修改，包含一項名為 Single Root I/O Virtualization (SR-IOV) 的功能，在嵌入於 RHEL 5.5 中的 Xen 或 KVM hypervisor 執行的 guest 作業系統，將可完全驅動並發揮 10 Gigabit Ethernet 介面卡。Burke 宣稱 RHEL 5.5 是目前唯一能達成此要求的 hypervisor 環境。

RHEL 5.5 更新提供了許多虛擬化技術增強。雖然不久前紅帽的 KVM hypervisor 商業實作 Red Hat Enterprise Virtualization (RHEV)，才剛剛推出採用 RHEL 5.5 核心的 RHEV 2.2 beta 版本。如今正式推出的 RHEL 5.5 支援了更加龐大的 guest 虛擬機器。RHEV 2.2 beta 針對各 guest 支援 16 顆虛擬 CPU 和多達 256 GB 記憶體，而 RHEL 5.5 可在一個 Xen 或 KVM guest 中，支援 32 個實體處理器核心以及多達 512 GB 記憶體。

RHEL 5.5 提供了微軟 (Microsoft) Windows 7 的相互運作性加強，並擴展 Active Directory 整合性，包括改善使用者與群組對應，和簡化跨平台檔案系統管理。

Burke 表示，他們相信 RHEL 持續推動該作業系統平台的進化，提供具備效能、可靠性、延展性、可負擔性的新技術給用戶。RHEL 提供了實體、虛擬、雲端部署的基礎，將持續作為許多人最苛刻關鍵任務的平台選擇。
