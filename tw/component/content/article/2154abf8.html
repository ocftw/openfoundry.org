紅帽釋出新一代 Java 平台瞄準雲端計算

建立日期 2009-09-13 18:10      最近更新在 2009-09-30 23:05 作者是 謝良奇/編譯

紅帽 (Red Hat) 宣佈推出 JBoss Enterprise Application Platform 5.0，作為該公司 JBoss Open Choice 策略中的核心中介軟體元件，該平台代表了新一代 Java 應用軟體平台，且將成為紅帽雲端基礎不可或缺的一部份。

JBoss Enterprise Application Platform 5.0 的設計針對多種 Java 程式設計與元件模型，提供了廣泛的支援，例如：Plain Old Java Objects、Java EE、Spring Framework、OSGi, Google Web Toolkit，以及各種多樣化網際網路應用軟體 (Rich Internet Application，RIA) 框架和動態語言。

此外，該平台跨越了數量漸增的各類運作模式，從傳統就地部署 (on-premise) 式的輕量級 Java 應用軟體，到新的內部與外部網雲部署。由於縮短應用軟體部署、開發，開發者在程式設計模型上的選擇，針對程式設計模型變更時的保障，以及更快速轉移至更新的應用軟體部署模型，用戶預料將可因此獲益。

創意行動與網站解決方案開發者 Sensei 資訊長 Tim Dion 表示，JBoss Enterprise Middleware 讓他們的架構得以極為彈性，能夠部署一系列選項以便應付他們的需求。由於雲端平台的加入，紅帽正在為該公司的未來需求作出預測，以便協助他們無痛地轉換到雲端計算環境。醫療照護創新與行動科技扮演的角色，正在改變個人與組織，像 JBoss 這樣的雲端計算環境有助將這些創新帶給群眾。

紅帽的中介軟體系列產品已經準備要協助用戶好好運用雲端計算。作為紅帽雲端計算基礎的核心元件，用戶將可結合 JBoss Enterprise Application Platform、Red Hat Enterprise Linux 與整合式的虛擬化技術，提供業界第一套完整虛擬化、基於標準的企業開放源碼雲端計算服務平台，支援就地與非就地的網雲部署。

JBoss Enterprise Application Platform 是第一套以商業化形式出現在 Amazon 的 Elastic Compute Cloud 上的 Java EE 應用程式伺服器。這次的新版釋出預料也將出現在 Amazon EC2，以及其他通過紅帽 Premier Cloud Provider Certification 的環境上。

最近 IDC 預測 IT 雲端服務上的支出將成長 3 倍，在 2012 年達到 420 億美元，佔 2012 年 IT 支出成長的 25%，以及接下來一年近 1/3 的成長。IDC 指出，成功的供應商將必須同時面臨雲端服務的最大挑戰，以及最大的傳統 IT 使用者議題。

紅帽中介軟體副總裁 Craig Muzilla 表示，他們不僅僅藉著釋出 JBoss Enterprise Application Platform 5.0 來發表其 JBoss Open Choice 策略，他們也對 Java 應用軟體平台從傳統走向雲端環境的轉變做出貢獻。最終他們希望給予其用戶選擇正確程式設計與部署模型的彈性，以便領先其競爭對手。

JBoss Enterprise Application Platform 5.0 的新功能包括新一代以服務為基礎的架構，核心的 JBoss Microcontainer 提供改善過的類別載入、效能、生命週期管理，與跨越 Java EE、POJOs、OSGi、Spring Framework 等多種程式設計與元件模型的彈性，Microcontainer 更提供劃分企業服務與核心執行時期引擎的能力，增加可組態性。

在改進的管理與組態功能上，嵌入式版本的 JBoss Operations Network 管理控制台提供包羅萬象的介面，以管理與設定應用軟體，控制應用程式伺服器運作，並提供有關就地與非就地應用軟體的效能指標。

此外，JBoss Enterprise Application Platform 提供了完整的 Java EE 5 支援，以及包括 Java EE Web Profile 概念等若干 Java EE 6 預定標準的支援。

主要元件升級包括 JBoss Application Server 企業版、Hibernate, Seam, JBoss Cache 與 JBoss Web Services，提供延展性的效能增強、高可用性與跨越多種部署環境的智慧型負載平衡。新版釋出更針對分散式交易與廣泛的網路服務堆疊提供支援。
