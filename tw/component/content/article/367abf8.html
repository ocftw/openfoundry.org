微軟、Novell 合作 開放源碼界憂心影響

建立日期 2006-11-04 08:00      最近更新在 2007-03-09 00:57 作者是 謝良奇
在微軟與 Linux 供應商 Novell 日前宣佈合作，讓雙方用戶能更輕易同時使用 Windows 與 Linux 軟體，更重要的是承諾微軟與 Novell 彼此不會向對方提起專利侵害訴訟，減輕雙方用戶心裡存在已久的潛在擔憂。然而，微軟與 Novell 這次的合作在社群內也掀起微軟是否正透過拉攏 Novell 進而分化 Linux 社群，甚至有人認為這個合作案將對開放源碼運動有負面影響。

繼日前 Oracle 宣佈為 Linux 散佈套件供應商紅帽 (Red Hat) 產品提供技術支援後，微軟與 Novell 簽訂的合作案，又為紅帽增添新的市場壓力。合作案中微軟與 Novell 將設置共同研發中心，供雙方設計與測試軟體整合，並與客戶及開發人員一同建構與支援這些技術。 微軟將推薦 Novell Suse Linux 企業版給有意使用 Windows 與 Linux 的用戶。微軟執行長 Steve Ballmer 表示，他們會明確告知有意同時採用 Windows 與 Linux 的用戶選用 Novell。微軟也將提供最少 7 萬份折價卷，給微軟的企業用戶，以換取 Suse Linux 的 1 年支援訂購。 然而，在這份合作案中，更重要的是，微軟與 Novell 協議保證未來不會以專利侵害控告 Novell 的 Linux 用戶。軟體專利訴訟一直以來，都是 Linux 企業用戶的潛在威脅，這方面的考量經常嚇退企業用戶。開放源碼開發實驗室執行長 Stuart Cohen 也表示，部份用戶因為擔憂會成為專利訴訟的對象，而放棄使用這類軟體。開放源碼專案 Wine 開發者 Tom Wickline 就警告，藉此微軟將控制自由軟體的商業使用。 Wickline 認為此一合作的背後意義在於，微軟承諾不對個別的非商業開發者提出專利訴訟，無形中塑造出微軟認可的開放源碼模式，不遵照這個模式的商業開放源碼將成為其專利訴訟的對象，藉此威脅不遵循微軟承認的 Novell 模式之外的開放源碼。換句話說，Wickline 認為 Novell 將成為下一個 SCO。 除了面臨微軟與Novell 結盟的壓力外，紅帽最近亦面對 Oracle 的競爭，不過曾於 2001 年訪問台灣的 Don Rosenberg 就認為 Oracle 等的做法難以憾動紅帽產品的市場地位。他認為 Oracle 針對紅帽產品提供服務，正是開放源碼可以免於被單一廠商套牢的情形，同時，儘管 Oracle 能夠從紅帽產品衍生出自己的產品，紅帽卻可以很輕鬆地把 Oracle 的程式碼整合到自己的產品中，透過開放源碼的特性，Oracle 無法簡單地衍生紅帽的產品而吸納其用戶。
