紅帽推出新 hypervisor 計畫於 Linux 內建虛擬機器管理功能

建立日期 2008-06-26 08:00      作者是 謝良奇/編譯
紅帽 (Red Hat) 日前在年度大會上宣佈為其 Linux 平台改善彈性、開放性與管理功能，將推出新的 hypervisor，並為 Red Hat Network (RHN) Linux 管理平台開放源碼。紅帽正計畫將虛擬機器管理功能建置於 Linux 作業系統中。

紅帽在 oVirt 專案上發表了以 Linux 為基礎的 hypervisor，嵌入式 Linux Hypervisor (Embedded Linux Hypervisor)，可置於 64MB 的 flash 磁碟中，於 x86 硬體上執行虛擬啟動。紅帽同時也釋出了 RHN Satellite 的原始碼。Satellite 是 Linux 的自動化管理工具，用來管理包括實體與虛擬伺服器的內容更新、系統更新與監測、權限與排程。
Linux 基金會 (Linux Foundation) 執行董事 Jim Zemlin 表示，RHN 開放源碼為 Spacewalk 專案，對開放源碼與 Linux 社群是一大福音。他說，這是相當重要的軟體，紅帽此舉顯示其做為開放源碼軟體公司的貢獻。

紅帽表示，隨著 Satellite 開放源碼，預期該專案將可在 Linux 系統管理上，催生出更多新的想法。紅帽企業管理副總裁 Katrinka McCallum 表示，客戶使用後表示 Satellite 能讓他們管理多達 4 倍數量的紅帽伺服器。系統環境擴展時，卻不需要增加人力。她說，未來 Spacewalk 專案將可做為 Satellite 上游的程式碼貢獻來源。

紅帽執行長 Jim Whitehurst 表示，他們認為透過開放源碼，該產品的成長會更快速，並且獲得使用社群。他們相信這是開發軟體的絕佳模式。Gartner 分析師 George Weiss 指出，除了獨立開發者，Oracle 與 Novell 等 Linux 散佈商也會因 RHN Satellite 開放源碼而獲益。

在其年度大會上，紅帽公開了 beta 版本的 hypervisor。紅帽前工程執行副總裁，現任產品與技術總裁  Paul Cormier 指出，紅帽將提供基於核心虛擬機器 (Kernel Virtual Machine，KVM) 的內建 hypervisor，以及可用以產生並管理數千個虛擬機器的管理工具。

KVM 於 2006 年已由 Linus Torvalds 加入核心與核心開發流程中，目前仍為初期採用階段，提供給用戶測試，可見於例如 Red Hat Fedora 等 Linux 散佈套件中。但並未納入紅帽目前的 Enterprise Linux 5 中。Zemlin 表示，由於 KVM 已進入主要 Linux 開發流程中，有助降低其支援門檻。

紅帽過去的虛擬技術策略，如 Oracle、Sun 等廠商，乃建構於開放源碼的 Xen 之上。Xen 目前已經是 Citrix 的產品。紅帽主要的企業 Linux 產品 Red Hat Enterprise Linux 5，整合了基於 Xen 的虛擬化技術。

對於各家廠商而言，Xen 成為 Citrix 產品後，依賴 Xen 開放源碼軟體做為自家虛擬化技術策略顯得過於被動，如今紅帽正有意利用 oVirt，重新取得虛擬化技術掌控權。儘管如此，這意味著紅帽必須讓虛擬化市場再接受另一套虛擬化技術。已經成為 Linux 作業系統核心的 KVM，自然成為紅帽最佳的整合對象。

除此之外，紅帽此舉相信將有助促使 Linux 成為虛擬機器管理的作業系統首選。紅帽 Red Hat Enterprise Linux 未來版本將擁有內建於核心的 hypervisor，以及一系列管理工具。

被問到有關與 Xen 的合作關係，是否將因推出自有的 hypervisor 而有所改變，紅帽高層則表示該公司將同時保有兩套技術，依照用戶管理虛擬工作所需，提供用戶不同選擇。

紅帽在虛擬化技術上的動作，也被視為針對虛擬化技術領導者 VMware 而來。Zemlin 甚至表示，紅帽的新 hypervisor，在安全性與管理性皆優於 VMware。

許多分析師長期而言並不看好，VMware 採用私有軟體且並不納入作業系統的模式。RedMonk 主分析師 Stephen O'Grady 就認為，就如作業系統與關聯式資料庫系統一般，開放源碼也將對虛擬化領域造成衝擊，為用戶降低成本並擺脫私有封閉情況。

儘管紅帽持續為其 Linux 散佈套件整合虛擬化技術，終端用戶是否買單目前仍有待觀察。日前 ZDNet UK 所作針對伺服器作業系統的態度研究顯示，許多用戶雖然關注虛擬化技術，但仍將其視為與作業系統獨立的技術。在作業系統功能的重要性順序上，許多受訪者將虛擬化技術列為最後的項目。

不管如何，紅帽正逐步重新掌握其虛擬化策略，儘管技術接受度，以及面對各家競爭對手，紅帽自家的技術能有幾分把握，目前仍不明朗，但 oVirt 可說是紅帽近期相當重要的策略動作。
