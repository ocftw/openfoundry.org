紅帽發表即時 Linux 擴充功能 協助企業 IT 架構部署即時運算能力

建立日期 2007-12-11 08:00      作者是 謝良奇/編譯
開放源碼公司紅帽（Red Hat）發表了在該公司 Linux 上的即時擴充功能。部份功能根據紅帽宣稱，比類似的競爭技術，要快上一百倍。即時 Linux 版本鎖定的是有可預測回應需求的應用軟體，例如程序控管或金融市場的交易應用。


紅帽的即時傳訊網格（Messaging Real-time Grid，MRG）以 beta 版本釋出，2008 上半年將釋出完整版本。MRG 是紅帽企業 Linux 平台（Red Hat Enterprise Linux）的延伸功能，專為銀行等需要 IT 系統即時完成交易的企業需求所設計。

MRG 包含了傳訊佇列中介軟體，基於先進傳訊佇列協定（Advanced Message Queuing Protocol，AMQP），讓應用軟體彼此溝通。MRG 是 AMQP 第一套商業實作。開放源碼系統 AMQP，根據紅帽副總裁 Scott Crenshaw 表示，比 Tibco 與 IBM 的私有競爭產品，要快上一百倍。

Crenshaw 認為這種效能上的大幅提升，還會有持續突破，並且將其歸功於社群開發模式。

Crenshaw 表示，MRG 使紅帽的核心作業系統有了即時能力。透過即時作業系統，IT 經理人將能確保系統反應時間，無須過度擴充系統只為了達到所需的服務層級。他們的預測，企業基礎架構將全面納入即時能力。

MRG 可說是紅帽上個月發表的自動化策略的首批成果。紅帽希望藉由該策略，為使用者與軟體廠商簡化企業基礎架構建構。紅帽 EMEA 區域副總裁 Werner Knoblich 表示，隨處都可以執行所有的應用程式，像是實體伺服器、虛擬伺服器、計算網雲（compute cloud）或應用裝置等。軟體廠商只需驗證一次，就可以確保軟體在隨處執行無誤。

MRG 平台另一項功能是網格能力。企業可以利用該公司從 Linux 桌面系統，「借用」閒置的處理器時間，並管理分散式工作負載，排定位於本地或遠端網格的工作。

Crenshaw 說，過去網格只應用在高效能運算上（high-performance computing，HPC），現在他們延伸到了所有的應用軟體，他們稱之為高生產力運算（high throughput computing，HTC）。例如，當網頁伺服器需求增加時，企業的資訊系統就要能夠運用桌面與商業網格服務的計算能力，例如 Amazon 的彈性計算網雲（Elastic Compute Cloud）。

這項分散式運算技術來自紅帽與威斯康辛大學（University of Wisconsin）的合作，以及其高生產力計算專案 Condor。從桌面系統借用處理器時間的技術，是基於和 Intel 的 vPro 桌面管理技術的整合，從而改進了紅帽企業 Linux 桌面系統管理能力。

MRG 可以執行在 Java、Solaris 與 .Net 平台上。根據紅帽的說法，目前以在紅帽企業上的效能最好。Crenshaw 表示這套系統第一位客戶是美國海軍（U.S. Navy）。他說，幾年前對方就曾為了船艦、武器控制系統跟他們接洽。
