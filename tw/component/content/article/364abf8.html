微軟與 Zend 共同合作改善 Windows PHP 環境

建立日期 2006-10-19 08:00      最近更新在 2007-03-09 00:59 作者是 謝良奇
微軟與 PHP 程式語言產品與服務供應商 Zend Technologies 日前發佈了共同合作的消息。開放源碼腳本語言 PHP 是 Web 上最常運用的程式語言之一，微軟這次與 Zend 攜手希望共同改善 Windows 上的 PHP 環境，期望能藉重 PHP 的領導地位，為 Windows 平台拉攏更多原本在 Linux、Unix 的 PHP 用戶。

這個消息在 Zend/PHP 大會上由微軟與 Zend 高層宣佈，Zend 共同創辦人兼技術副總裁 Andi Gutmans 表示，Zend 計劃將合作中的技術改進整合到 Zend 未來的產品線。微軟與 Zend 在這個技術合作案上，希望能提供用戶功能更加完善、整合性更高的 Windows PHP 環境。 微軟與 Zend 的合作將在 Windows Server（Longhorn）、Windows XP、Windows Server 2003 與 Vista 上，提供產品級的 PHP 執行環境。微軟提供 FastCGI 支援，讓 Windows 2003 這類的多執行緒作業系統一次連接多個 PHP 程序，而不是一次執行一個 PHP 程式。 PHP 根據 Evans Data Corp. 的調查，以 350 萬使用者列為 Web 上最受歡迎的程式語言，由於效能上的因素，使用 PHP 的網站多半搭配開放源碼的 Apache 伺服器與 Linux 而非微軟 Windows Information Server 與 Windows 作業系統。Gutmans 表示，微軟與 Zend 希望彌補這部份的差距，Zend 將會最佳化 Windows 上的 PHP，讓過去透過 POSIX APIs 與 Windows 互動的 PHP，能直接辨識 Windows API。他說，Facebook、Flickr 與 Yahoo 都大量使用 PHP。甚至有 75% 的 Web 2.0 網站都是 PHP 的用戶。 網路調查公司 Netcraft 指出，微軟 Information Server 近年相對於 Apache 伺服器，搶佔了不少市場，由原先的 20% 弱上升到 31.4%。儘管微軟的主要平台 Windows Information Server 上本來就有自家的程式語言 Active Server Pages，對於希望進一步拉抬 Information Server 甚至是 Windows 作業系統平台的微軟，PHP 背後龐大的使用者族群絕對不容忽視。 Burton Group 資深分析師 Richard Monson-Haefel 就認為，Java、Cold Fusion 與 .Net 等開發工具與 PHP 族群相比，規模實在相形見拙，這一點對微軟勢必極為重要，微軟理解到如果希望擴大其平台的普及率，只支援 .Net 是不夠的。 Interarbor Solutions 首席分析師 Dana Gardner 看好這次的合作，他認為微軟藉此可擁抱開放標準。微軟近來接連地接納更多開放源碼解決方案，如 Xen 虛擬技術、SugarCRM，似乎企圖尋求其私有平台與開放源碼軟體共存之道。Gardner 表示，這麼一來用戶與開發者可以得到更多選擇。
