Sun 將打造 Solaris 成為下一個 Linux

建立日期 2007-09-04 16:00      最近更新在 2007-09-17 10:25 作者是 謝良奇/編譯

Sun Microsystems 正計畫把從 Linux 學來的一切用在 Solaris 上，要讓 Solaris 作業系統的商業與開放源碼版本，像 Java 一樣成為隨處可見的技術平台。

Sun 的 Java 程式語言在 1995 年就推出，如今在多數電腦、行動裝置與嵌入式系統都有 Java 的蹤影。Sun 想將這個經驗複製到該公司的 Unix 品牌 Solaris 身上。為了達到這個目標，首先要做的就是廣納開發者支持，也要設法粉碎長久以來外界對這套作業系統的偏見。

Sun 的 Solaris 行銷副總裁 Marc Hamilton 表示，Sun 希望把這套作業系統帶到桌面、嵌入式系統等過去並非 Solaris 擅長的市場。換言之，Sun 的目標是將 Solaris 定位成 Linux 的替代系統。

Sun 作業平台長 Ian Murdock 認為 Solaris 背後蘊釀的氣勢相當可觀。Murdock 過去曾擔任 Linux 基金會（Linux Foundation）的技術長，同時也是 Debian Linux 散佈套件的創始人，Murdock 過去對 Linux 社群的經驗，的確有助 Sun 對 Solaris 的發展計劃推動。

Sun 明年初將會釋出代號為 Indiana 專案的 OpenSolaris 執行版本，預料將與 Linux 散佈套件相似。該專案將會有包含基本作業系統與桌面環境的安裝 CD，開發者可選擇從網路安裝附加軟體。

Murdock 表示 Indiana 的目標是為 Solaris 建立開發者社群。因此 Indiana 也將加速釋出週期，每 6 個月將推出一個新版本。Murdock 說，Sun 會在 Indiana 上採用兩層式的開發模式，為開發者與先期試用者制定出新技術從 Indiana 與 OpenSolaris，進入保守的企業用戶所使用的 Solaris 的規則。對此的主要挑戰，Murdock 認為，是如何為兩個相當不同的社群，有效率提供單一 Solaris 平台。

此外，讓開發者對 Solaris 背後的技術產生興趣，是 Sun 另一個課題，特別是要吸引對這個作業系統有過負面經驗的人。

Solaris 的成功因素不僅僅需要開發者，Sun 正在設法增加這套作業系統支援的硬體裝置。這個月初 Sun 與該公司的主要硬體競爭對手 IBM 達成了一項過去難以想像的合作協議。IBM 同意在部份 System x 與 BladeCenter 伺服器上提供 Solaris 作業系統與 Solaris 服務訂購 （Solaris Subscriptions）。IBM 一來也算是為使用者提供多一種作業系統選擇，一方面也可在支援服務上增加收益。

Hamilton 表示 Sun 與約 40 家 OEM 廠商正在討論 Solaris 的硬體支援。這些公司包含了只在特定地區營業的小硬體廠，不過 Hamilton 也希望能與 HP、Dell 達成如同和 IBM 的合作關係。
