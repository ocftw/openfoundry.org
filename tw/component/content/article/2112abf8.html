紅帽推出雲端運算供應商認證計畫

建立日期 2009-07-02 08:00      作者是 謝良奇/編譯
為了確保其企業 Linux 與 JBoss 軟體得以作為雲端運算架構的核心元件，以及基於紅帽的應用軟體能在網雲上安全可靠地執行，紅帽 (Red Hat) 日前啟動新的夥伴計畫。根據紅帽表示，新的 Premier Cloud Provider Certification and Partner Program 將針對雲端運算供應商進行認證，提供應用軟體，以及基於包括 RHEL (Red Hat Enterprise Linux)、JBoss Java 中介軟體等紅帽軟體的架構。

紅帽正在嘗試在網雲上讓應用軟體，能夠更輕易地在 Linux 底下運作。紅帽組織發展副總裁 Michael Evans 在近期訪談中指出，80 到 90% 的網雲靠的是開放源碼技術。RHEL 與 JBoss 中介軟體的使用正是這些開放源碼程式碼的最佳搭配。Evans 表示，紅帽發現該公司的客戶希望，紅帽的產品訂用能夠延伸應用到外部的網雲供應商。

這項計畫的第一個合作對象是 Amazon。Amazon Web Services 早已是該公司技術夥伴，在 Elastic Compute Cloud (EC2) 上提供 RHEL 運行選項。Amazon 將與紅帽合作，確保希望在 EC2 上執行其軟體的應用軟體供應商，能夠針對 RHEL 與應用軟體的組合，產生正確的虛擬化 AMI (Amazon Machine Image) 格式。

Evans 表示，紅帽考量到雲端運算的不同使用顧客，如終端使用者與獨立軟體廠商，決定設置雲端運算供應商的合作計畫以提供服務。他表示，不同的紅帽用戶對於網雲有不同的關注層面與需求。企業用戶希望在其資料中心紅帽系統上執行的應用軟體，可以在網雲上可靠安全地運作。獨立軟體商則要確保其應用軟體，可以輕鬆地延伸到網雲上。

Evans 認為，透過與獨立應用軟體供應商的合作，該公司可以確保與特定應用軟體一起封裝並轉換為 AMI 檔案的 RHEL，可以在 Amazon 的 EC2 網雲上執行。同時，紅帽與應用軟體供應商，也能夠根據使用小時、週數或月數而制定價格，以適應使用者的雲端運算形態。

儘管紅帽公開揭露此一計畫，但在 8 月前將不會公開計畫的詳細規定。Evans 說，屆時該公司也會公佈其他合作夥伴。

透過此一計畫，紅帽將與雲端運算架構供應商合作，在技術上確保用戶能夠將其 RHEL 與 JBoss 的產品定用移動至網雲。該公司也將在技術上啟用按時計算、隨用隨付的 RHEL 與 JBoss 版本，並且與雲端運算供應商聯合提供技術支援。紅帽希望其用戶在移動其應用軟體之後，將能自紅帽獲得和移動前相同等級的支援。Evans 表示，紅帽也計畫與合作夥伴進行協同行銷與銷售。

目前雲端運算市場僅有數家大型廠商，如 Google、Rackspace、Verizon、IBM、Salesforce.com 與微軟，Evans 預估該市場最終將出現 50 到 100 家雲端運算供應商。他認為這是雲端運算的放牧期，就像是未加開發的荒野西部，該公司希望為此市場帶來可靠性與安全性，並給予用戶更多選擇。

由於雲端運算供應市場仍未底定，此次針對網雲的應用軟體轉移業務，對於紅帽仍屬於部署的早期階段。儘管經濟情勢不佳減緩了雲端運算的發展，分析師預期以網雲為基礎的 IT 服務市場，將持續成長。研究機構 IDC 預測，到了 2012 年，以網雲為基礎的 IT 服務支出將達 420 億美元，佔該年 IT 支出成長的 25%。
