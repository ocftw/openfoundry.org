OSDL 提 Portland 整合桌面介面專案

建立日期 2006-04-07 08:00      最近更新在 2007-03-08 08:02 作者是 謝良奇
開放源碼開發實驗室 (Open Source Development Labs, OSDL) 與 freedesktop.org 於波士頓舉行的 LinuxWorld 大會推出整合 GNOME、KDE 的共通界面－Portland 專案。未來開發者只要寫一次程式碼，就能在 KDE、GNOME 桌面環境上執行。 這次公開的 Portland 專案技術預覽版有兩大界面，包括一組命令列工具以及一組 API 程式庫名為 DAPI。Linux 開發者透過 DAPI 就可以提供客製化服務，同時維持跨平台的共通界面。開發者將不需要在撰寫軟體時，被迫在兩套界面間做出抉擇，

Portland 專案的誕生主要源自 OSDL 於去年底在波特蘭 (Portland)舉行的桌面設計師大會，其任務在於大幅簡化移植與整合 GNOME、KDE 兩大 Linux 桌面應用軟體的流程。

OSDL 執行長 Stuart Cohen 表示，Portland 專案應該能給桌面 Linux 的發展注入強心針。根據 Cohen 的說法，目前桌面 Linux 的市場佔有率僅約 4% 到 5%。該專案預料將能加速開放源碼作業系統在桌面環境的接受度。

這套新的共通界面已經釋出給獨立軟體廠供測試之用，5 月將釋出第一個 beta 版本，Portland 1.0 預計會在 6 月問世。最終版本將會納入 Linux產業標準 Linux 標準基礎 (Linux Standard Base) 中。

Intel 的 Linux 客戶端設計師 Waldo Bastian 表示，Portland 專案的釋出時間表有賴於社群、Linux 廠商和獨立軟體商的合作與持續投入。以過去 3 個月來所見各方的努力，時間表將能按時達成，獨立軟體商對於預覽版本的回饋會讓 1.0 版本更加成熟。

GNOME 基金會董事會成員之一的 Jeff Waugh 認為，他們相信 Portland 專案案會擴大 Linux 桌面版圖，從而拉抬對 GNOME 軟體的需求。該專案會進一步促使第三方為 Linux 桌面開發企業應用軟體。

KDE 技術經理 Thiago Macieira 則表示，獨立軟體商最終將可透過共通界面，撰寫出能整合各類作業環境的應用軟體，KDE 正在和獨立軟體商緊密合作，朝此一有助 Linux 桌面整體環境的方向前進。

獨立軟體商 CodeWeavers 執行長 Jeremy White 認為，Portland 專案可說是獨立軟體商的一大福音，一來可簡化基本的桌面整合工作、縮短開發時程，二來將可允許獨立軟體商推出可整合不同用戶環境的單一產品。此外，這組新 API 也能給予桌面電腦製造商如 Dell、HP，以及企業應用軟體公司如 SAP，更多支援桌面 Linux 的動機。
