支持桌面 Linux 易用性改善 Ubuntu 出資贊助開發者

建立日期 2008-09-23 08:00      作者是 謝良奇/編譯
Ubuntu 商業贊助商 Canonical 執行長 Mark Shuttleworth 近來曾談及，桌面 Linux 將在易用性與美觀性上，將逐漸接近 Mac OS X。做而言不如起而行，如今，他以及 Canonical 似乎準備就緒，展開實際贊助開發者的行動。Shuttleworth 的 Ubuntu Linux  專案為了將 Linux 打造成世界上最容易使用的作業系統，已經聘請由設計者、使用者經驗專家與互動設計專家組成團隊。

Shuttleworth 日前宣佈代號 Jaunty Jackalope 的下一個 Ubuntu 釋出版本 9.04 版時，提到了 Ubuntu 的願景。他表示，他們要將整個開放源碼圈子最優良的想法，都呈現在 Ubuntu 中。Ubuntu 的社群經理 Jono Bacon 則提到，Ubuntu 背後的贊助商 Canonical，將會實際贊助希望改善 Ubuntu 的開發者。

對此一艱鉅挑戰，Shuttleworth 承認他沒有最後解答。他表示，Ubuntu 新的上游團隊，將會與其平台團隊個別運作，以避免不必要的衝突。該團隊一開始將會把重心擺在參與 X、 OpenGL、GTK、Qt、GNOME、KDE 等專案上。

此外，Ubuntu 對於這些 Linux 介面軟體領導專案的參與，將落實在實際的程式開發工作上。這些成果將置於 Ubuntu 的 LaunchPad 帳號下的 BZR (Bazaar Version Control) 分支中。Shuttleworth 指出，如果他們僅僅秀出一些圖片或系統雛形，就要求對方按照這些方式重新更改其專案，肯定不會有多大效果。

根據 SDTimes 的報導，紅帽 (Red Hat) 仍是 Linux 核心的首要企業貢獻者。近日 Novell 員工也是知名的 Linux 核心開發者 Greg Kroah-Hartman 公開讚揚紅帽的貢獻，也大肆批評 Canonical 的貢獻不足。

對此，Shuttleworth 出面辯護，他說，他們所作的大部分都著重在整合上。他們的競爭者將此視為，Canonical 沒有貢獻。然而，更精確地應該說，他們的貢獻在於他們是如何有效地將上游的最新穩定成果，加上安全性的維護，儘最大可能讓更多使用者來測試或使用。對他而言，Shuttleworth 說，這是很大的貢獻。

目前為止，除了紅帽之外，Novell 也是 Linux 桌面的主要上游貢獻者。然而，以上廠商主要焦點放在企業 Linux 桌面，對照 Shuttleworth 的想法，或許這些廠商的企圖與動機更為功利主義取向。

Canonical 對於桌面 Linux 的易用性貢獻良多，如今除了過去整合上游開發者的成果，該公司將投入更多資源。Shuttleworth 表示，逐漸地，Canonical 將扮演起，推動 Ubuntu 之中軟體真正改變的角色。

Mac 近來逐漸可見於企業辦公室場合，原因在於 Mac 創造一群忠實的消費者，堅持將 Mac 的美感與可用性帶到企業中。桌面 Linux 如果要成功，就必須創造更愉快的使用經驗，讓使用者想要使用桌面 Linux。Shuttleworth 很了解這點，這也是 Canonical 正確的下一步。
