紅帽釋出 SPICE 虛擬桌面協定 推動桌面虛擬化技術普及

建立日期 2009-12-17 08:00      作者是 謝良奇/編譯

開放源碼解決方案供應商紅帽 (Red Hat) 宣佈，為了與合作夥伴開放協同合作以推動虛擬技術，該公司決定釋出 SPICE (Simple Protocol for Independent Computing Environment) 代管虛擬桌面協定 (hosted virtual desktop protocol)。

紅帽很早就開始擁抱虛擬化市場，一開始是讓開放源碼 Xen 技術成為其 Enterprise Linux 作業系統中的一部份。為了在虛擬化市場扮演更為關鍵的角色，紅帽從 Xen 轉向支持 KVM hypervisor，並在 Red Hat Enterprise Linux (RHEL) 中嵌入該技術。下一步是在 2008 年 9 月購併 Qumranet，藉此取得多項關鍵技術，如 KVM、SolidICE 桌面虛擬平台，以及 SPICE 遠端協定。

SPICE 是微軟 Remote Desktop Protocol (RDP) 的替代方案，被用來將 PC 連接到遠端桌面與伺服器。根據紅帽表示，SPICE 是目前處於 beta 階段的 Red Hat Enterprise Virtualization for Desktops 產品的核心元件。透過 Spice 專案，紅帽將與合作夥伴以及開放源碼社群協同合作，擴展此一協定的開發，為的是打破虛擬化技術普及的障礙。

作為紅帽購併 Qumranet 取得的元件，SPICE 是一套專為虛擬環境設計的適應性遠端 rendering 協定。藉由其現代化的通訊技術，SPICE 的目標是為講求頻寬的應用軟體，如多媒體與 VoIP 等，提供與實體桌面匹敵的使用者體驗。

由於遠端協定是打入桌面虛擬化的關鍵元件之一，各家廠商莫不爭相加強與更新現有遠端技術，以贏得虛擬化技術的終端使用者認可。例如 Citrix 提出 ICA/HDX，微軟更新其 RDP 並且希望以購併自 Calista 的技術，進一步提升其技術，VMware 則正投入 Teradici 的 PCoIP 技術。

紅帽技術長兼工程副總裁 Brian Stevens 表示，透過開放源碼 SPICE 技術，該公司的產業夥伴與社群能夠與該公司共同對虛擬化的未來做出貢獻。SPICE 協定的設計是透過自動調適於所處的圖形與通訊環境，進而達到最佳化效能，因此廠商擁有相當大的機會，針對各自特殊的應用，增強該技術。

The 451 Group 企業軟體分析師 Jay Lyman 指出，盡管將購併來的技術加以開放源碼，對紅帽來說是司空見慣，該公司更進一步支援實作程式碼，以及伺服器、用戶端部署的元件，以支持 SPICE 協定原始碼的開放源碼釋出。Lyman 表示，在虛擬化桌面技術與標準的採用上，我們仍處於早期階段，但開放源碼 SPICE 有潛力建構出開發者與產業圈支持，並作為不同 VDI 選項間的共同層。

IBM 開放系統開發副總裁 Daniel Frye 表示，紅帽釋出 SPICE 開放源碼協定，有助朝向產業異質性虛擬化解決方案的共同運作性目標邁進。IBM 希望與紅帽以及開放源碼社群合作，推動虛擬化技術的普及。

紅帽合理的下一步，應該是嘗試讓該協定成為 Linux 核心的一部份，就如 Qumranet 對 KVM hypervisor 技術所作的一樣。一旦如此，每個 Linux 散佈套件都將提供 SPICE 作為立即可用的方案選擇。SPICE 是 Red Hat Enterprise Virtualization for Desktops 包含的 3 項主要技術元件之一，這套解決方案預料將於 2010 推出。
