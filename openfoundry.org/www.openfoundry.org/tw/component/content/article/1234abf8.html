Adobe 釋出 Flex 開發工具 Linux 版本 搶占網站互動應用軟體開發市場

建立日期 2007-10-09 16:00      作者是 謝良奇/編譯
在微軟與 Novell 宣佈 Silverlight 開發工具 Linux 版本 Moonlight 的消息後，Adobe 很快地趕在日前舉行的 MAX 2007 會議上，釋出了 Flex Builder Linux 的 alpha 版本。這也是該公司第一個在 Linux 桌面上推出的 IDE。

Flex Builder Linux 是 Flex Builder 針對 Linux 桌面的第一個釋出版本。負責這個 Linux IDE 開發的 Flex 開發者 CH Swaroop 表示，由 Adobe Lab 釋出的這個 Eclipse 延伸套件有來自 Adobe 的官方支援。

這個版本包含了代號 Moxie 的 Flex 3 的部份功能，如建立專案、程式碼彩色編輯、程式碼提示、編譯、Flex 的 Ajax Bridge 與除錯。

Flex 是 Adobo 用來開發互動式網站應用軟體 (rich Internet applications) 的軟體開發堆疊，以 Eclipse 為基礎，與包括 Flash 在內的多項 Adobe 多媒體技術有密切關連。今年 4 月，Adobe 宣佈 Flex 開放源碼的計畫，但 Flex Builder IDE 並不在開放源碼的軟體之列。Flex 核心框架的 Java 程式碼、ActionScript 除錯器與 ActionScript 程式庫會用 Mozilla 公眾授權釋出 (Mozilla Public License)。

Adobe 是在 MAX 使用者大會上宣佈了 Flex Builder Linux 的消息。會上公佈的新技術還有用來建構網路應用軟體使用界面的設計工具 Thermo，與新的 Flex Builder 3 beta 版本，以及該公司新用戶端執行環境 Adobe AIR 的第二個 beta 版本。

Adobe 尚未宣佈 Flex Builder Linux 的完成日期或相關計畫，預料 2008 初才會有更詳細資訊。約一個月前，微軟與 Novell 支持的 Mono 專案才剛剛宣佈，共同將微軟的網站開發工具 Silverlight，移植到 Linux 平台。

Adobe 在 Flash Player Linux 版本上一度進展緩慢，引來 Linux 用戶不滿，今年終於釋出 Linux 版本的 Flash Player 9，算是落實了讓 Linux 趕上 Windows、Mac，這個 Adobe 一直以來對外宣稱的目標。

Flex Builder Linux 需要 FlashPlayer 9 的 Linux 版與 Firefox 及 Eclipse 3.3 才能執行，支援了 32 位元的 SuSE Linux 企業伺服器 10（SuSE Linux Enterprise Server 10）、紅帽企業 Linux 工作站 4（Red Hat Enterprise Linux Workstation 4） 與剛釋出的 Ubuntu 7.0.4。

開發者現已在 Adobe 論壇 (Adobe Forum) 提供建議，其中傳出要求 Adobe 提供開發源碼開發模式與雙重授權策略的聲音。例如署名 Nelson Batista 的使用者表示，希望能推出免費社群版本以及供付費使用的更完整版本，有機會吸引負擔不起 Silverlight 500 美元 IDE 的業餘開發者，而公司可以選擇付費授權，取得更完整的工具。
