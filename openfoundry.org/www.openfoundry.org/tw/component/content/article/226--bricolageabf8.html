從私有到開放－內容管理系統 Bricolage

建立日期 2005-04-29 08:00      最近更新在 2011-01-11 18:18 作者是 謝良奇
從 Salon 雜誌的內部計畫，到著名的開放源碼專案，Bricolage 在 SourceForge.net 中雖然其規模只能算是小型專案，但 Bricolage 卻廣受大型組織採用。Bricolage 從一項失敗計畫，轉而成為成功的 開放源碼專案，可說是開放源碼延續私有軟體成長的最好範例。 Bricolage 是一套開放源碼的企業級內容管理系統，能大幅簡化組織 內容建構、管理與發佈上的煩瑣事項。具備了可自定工作流程、自訂 文件形態、多網站管理機能與豐富的 Perl 模板支援，Bricolage 能 符合大型組織需要。eWeek 在 2002 年稱 Bricolage 為「印象最深 刻產品」。

Bricolage 的用戶包括了 Macworld、世界衛生組織（World Health Organization）、Portugal Telecom 等。Bricolage 原先為 Salon 雜誌的內部專案，該專案並未完成，後來由 About.com 接手。Wheeler 與該公司聯手開發出 Bricolage 1.0，該公司並決定將 Bricolage 開放源碼，維護者則由 Wheeler 自願擔任。Wheeler 並於數月後創辦 Kineticode 公司，負責銷售 Bricolage 諮詢、軟體訓練與支援等。

Perl 是 Bricolage 的開發語言，Bricolage 並支援 HTML mason、 HTML template 與 Template Toolkit 三種模板架構，但由於用戶提 出 PHP 模板需求，Wheeler 正與 PHP 社群合作發起 PHP Sandwich 專案，該專案可在 Perl 內載入 PHP 直譯器。與目前以 Perl 實作 的 PHP 直譯器不同，該直譯器能存取 Perl 空間的資料，並使用 Perl 載入的模組。

對 Bricolage 而言，未來將整合新的 PHP 模板架構，模板開發者能 夠用 PHP 撰寫模板。PHP 社群也因此受惠，得以立即取用 CPAN 上 的龐大模組。該項功能將能吸引 PHP 社群的目光，同時促進兩社群 間的交流。
