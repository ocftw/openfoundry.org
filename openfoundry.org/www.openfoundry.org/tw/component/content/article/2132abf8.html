Novell 推出 Linux 裝置計畫 降低獨立軟體商開發成本

建立日期 2009-07-30 08:00      作者是 謝良奇/編譯

Novell 推出 SUSE Appliance Program，協助獨立軟體廠商開發基於 Linux 裝置，其中包含 Amazon Elastic Compute Cloud (EC2) 系統上的 SUSE Linux Enterprise 組態設定。Novell 高層表示，該計畫能降低獨立軟體商的開發上的支出，並減少其客戶支援成本。

Novell 資深副總裁兼任行銷長 John Dragoon 表示，獨立軟體廠商藉由該項計畫，將更容易透過解決方案供應商銷售 Linux 為基礎的裝置。他說，在此該公司的價值，是作為獨立軟體廠商與解決方案供應商之間的撮合者。Dragoon 表示，Novell 將收取裝置所得營收的一小部份，而非向獨立軟體廠商先行收取費用。

Dragoon 指出，先行組態設定的裝置較容易經由散佈商銷售給解決方案供應商。Novell 已經與散佈商 Tech Data 達成合作支持此一計畫。Tech Data 軟體產品行銷副總裁 Stacy Nethercoat 表示，Novell 的裝置計畫會為 SUSE Linux 吸引新的經銷商，並有助將該技術的銷售擴展至資料中心之外。她表示，加值經銷商也較容易支援 Linux 裝置。

計畫中包含一系列新的技術，如 SUSE Studio Online，以及 Novell 的可客製化 SUSE Linux 裝置版本。SUSE Studio Online 是一套免費的 Web 工具，提供快速組合軟體裝置與客製化作業系統。

正如同微軟以 Visual Studio 等工具協助一般開發者提高生產力，以有效降低 IT 開發門檻。Novell 的 SUSE Studio Online 代表了線上開發軟體與虛擬裝置的嶄新方式，允許獨立軟體廠商客製化獲得完全支援的關鍵任務作業系統，並建構與測試完整的軟體堆疊。

Novell 並非唯一一家試圖簡化 Linux 技術的 Linux 廠商。Canonical 在打造直覺與易用 Linux 桌面上，下了很大工夫。紅帽 (Red Hat) 則致力於促進以 Linux 為基礎的虛擬化技術與雲端運算。

關於 Novell 的策略與其他 Linux 廠商不同之處，負責 SUSE Appliance Program 的 Novell 資深計畫經理 Matt Richards 指出，Novell 是唯一著重於協助獨立軟體商，從軟體裝置上獲利的作業系統廠商。SUSE Appliance Program 特別為了獨立軟體商而設計，提供獨立軟體商因應裝置建構、上市與擴展市場，所需之技術、銷售與行銷能力。Novell 也是唯一可讓獨立軟體商客製化關鍵任務作業系統，並取得完全支援的廠商。

Richards 表示，當這項技術提供給其散佈夥伴如 Tech Data，Novell 將提供獨特的價值訴求，協助獨立軟體商擴展其市場機會。

Novell 也支援了部署在 Amazon EC2 上的 SUSE Linux Enterprise Server 11。Dragoon 表示，這項服務將讓獨立軟體商在網雲上部署虛擬裝置。Novell 也提供了明年將推出的 SUSE Appliance Toolkit 的技術預覽，其中包含裝置廠商可用以發佈更新的 SUSE Lifecycle Management Server，以及基於 Web 的遠端組態工具 WebYaST。
