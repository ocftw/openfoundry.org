美大學院校以 Asterisk 平台取代 Cisco VoIP

建立日期 2006-09-20 08:00      最近更新在 2007-03-09 00:09 作者是 謝良奇
開放源碼 IP-PBX 平台 Asterisk 即將邁入 1.4 重大釋出版本，功能與效能皆有所提升，日前美國德州山姆休士頓州立大學 (Sam Houston State University, SHSU) 更已經率先進行計劃，將學校內的 Cisco VoIP 平台，全數更換為 Asterisk VoIP 網路。 SHSU 目前正把校內 6000 名教職員使用的 Cisco CallManager IP PBX 與舊有的 Nortel Meridian PBX 替換掉，改為用 Linux 伺服器執行 Asterisk。服務範圍包括電話、語音郵件、PSTN 閘道等。SHSU 的資深語音分析師 Aaron Daniel 表示，成本是主要考量。

Daniel 說，由於 Cisco CallManager 需要高額授權費，長期而言開放源碼方案可以節省許多成本。Cisco 的授權模式要求 CallManager 接的每部電話都需要各別授權。採用 Asterisk 連接既有 Cisco 電話的模式可以大幅降低授權費用。

SHSU 已經將 1600 部 IP 電話由 Cisco CallManagers 移至 Asterisk。該校採用 6 部 Dell 伺服器執行 Asterisk 平台，2 部做為 PSTN 閘道，2 部負責撥話處理，其它則提供語音郵件。

Mark Spencer 發起的 Asterisk 是首套開放源碼 PBX。Spencer 更於 1999 年成立 Digium 公司提供 Asterisk 服務與 Asterisk 企業版 (Asterisk Business Edition)。Digium 甫於日前由 Network World 選為自由軟體界最值得關注的 10 大公司之一。

Asterisk 在 1.0 版推出時便已在電信界引起不小震撼，如今更即將推出 1.4 版與硬體設備 Asterisk Appliance。此一重大釋出版本將新增 Jabber、Jingle 與 GoogleTalk 相容能力。Asterisk 用戶可望不久後與基於 Jabber/XMPP 的即時傳訊網路結合。Asterisk 1.4 也將新增 IPFAX 以便整合傳真、電子郵件和語音郵件。

除了成本之外，提高對 IP PBX 軟體與伺服器的控制，也是 SHSU 選擇 Asterisk 的一大理由。Daniel 表示，只有 Cisco 通過的伺服器更新與修補才能裝在以 Windows Server 2000 為基礎的 CallManagers 上。他們相當擔心駭客的問題。開放源碼有問題可以自行修正的特性，則讓他們安心許多。

另外，SHSU 在轉移到開放源碼 VoIP 的同時，代表也失去了 Cisco 的技術支援。所幸，Daniel 表示，他們透過郵件論壇與發展 Asterisk 的線上社群的幫助，目前為止還沒有遇到無法解決的問題。
