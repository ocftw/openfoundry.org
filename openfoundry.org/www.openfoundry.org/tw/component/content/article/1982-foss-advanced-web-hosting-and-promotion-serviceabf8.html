應用典範服務

建立日期 2009-02-11 01:15      最近更新在 2012-07-10 11:16 作者是 webmaster

為了增強國內自由軟體的應用幅度及活絡社群發展，自由軟體鑄造場本著服務公眾的精神，藉由『應用典範』的相關服務，提供更多表演舞台予國內的開放源碼(Open Source)專案及開放內容(Open Content)網站，透過服務社群的方式，達到促進台灣自由軟體社群發展的目標。因此希望在善用有限資源的情形下，提供自由軟體社群接近企業級的虛擬伺服器服務，希望可納入更多具備潛力之自由軟體社群專案，共同為台灣自由軟體社群發展盡一份心力。

服務內容：鑄造場提供下列服務

  • 提供硬碟空間及頻寬(視專案實際所需空間進行評估)
  • 舉辦自由軟體相關推廣活動時，得申請鑄造場協助支援(視活動向性及鑄造場年度工作負載進行評估)。

聲請評選條件

  • 成功或具有潛力之團體／社群
  • 專案內容為開放源碼軟體或開放內容的資料（Open Source Software or Open Content Material）
  • 專案內容為非商業性
  • 須推派一位代表人擔任固定連絡窗口
  • 需配合下列事項
    1. 必須在網站首頁放置鑄造場所提供之識別標誌（簡稱標誌），並設定連結至鑄造場首頁。標誌圖樣與連結設定事項請參考鑄造場標誌使用規範說明頁面中的「Powered by OSSF（網頁使用）」部分，網址為：https://www.openfoundry.org/OSSF-Logo.html/
    2. 連結收集網頁必須有鑄造場之連結：若該專案在網站上設有連結收集網頁，鑄造場的連結也必須出現在這個頁面。
    3. 在鑄造場Who’s Who（https://whoswho.openfoundry.org/）上登錄成為社群。
    4. 應鑄造場要求，提供可資證明專案活動狀況的資料，例如MRTG圖表、瀏覽點閱率或Google Analytics等雙方認可之數據資料。

徵選辦法

  • 提供資料　（範例 － Mozilla Taiwan )
    1. 專案簡介
      簡介專案目的與型態
    2. 專案歷史
      簡介專案發展緣起與現況
    3. 社群組成
      簡介參與專案之主要成員組成
    4. 資源需求
      提供預期需求的硬碟、頻寬與運算資源之評估

  • 評選過程
    1. 將專案相關資訊寄至 這個 E-mail 地址已經被防止灌水惡意程式保護，您需要啟用 Java Script 才能觀看 提報申請自由軟體鑄造場應用典範
    2. 經過初步資料審核並評估系統資源是否充足後，鑄造場將進行內部會議進行投票決議。
    3. 與決議通過之專案申請者連繫，簽訂應用典範契約並開啟相關資源。
    4. 將通過決議之專案簡介公佈於自由軟體鑄造場網站上成為應用典範。
