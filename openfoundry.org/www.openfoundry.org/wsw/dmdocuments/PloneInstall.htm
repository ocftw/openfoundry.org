Plone Installation

By marr, Last updated Mar 13, 2008

本文件介紹 Plone 系統的安裝與啟動技巧，協助讀者快速建立一個穩健而具擴充彈性的 CMS 系統。

開發大型 Plone 系統時，可以同時執行多個 Zope instance 並設定成「線上系統」(production server) 與「開發環境」(development server) 兩種模式，前者可以在系統線上運作，同時進行備份或小規模的測試，後者可以即時反應程式修改的結果，加速開發流程。

佈署開發環境

完整的開發環境，應盡量滿足程式員的需求，包括跨平台的合作環境，豐富的開發介面選擇，程式碼測試除錯流程的自動化，效能調校的便利性等。想要快速獲得 Plone 開發環境，最具效益的方式，在 Linux 環境是下載 Plone 的 Unified Installer 來編譯系統，在 Windows 環境是下載 Windows Installer 來安裝。

Linux 環境的編譯

編譯時需要的基本工具包括：

  • gcc : GNU C Compiler
  • gcc-c++ : GNU C++ Compiler
  • make : GNU make
  • tar : GNU tar
  • bash : GNU bash

其他建議的函式庫，及其相對應的 Linux [1] 包裝檔案慣例名稱，包括：

  • zlib :
  • jpeglib :
  • libxml2 : libxml2, libxml2-devel, libxml2-python, libxml2-utils
  • readline : libreadline5, libreadline5-devel, libncurses-devel [2]
  • libssl : openssl, libopenssl, libopenssl-devel
  • wv : wv, libwv, libwv-devel
  • xpdf : xpdf
  • odbc : libunixODBC1-devel, pyodbc, ZPyODBCDA

引入 libxml2 目的是讓 Plone 結合 marshall [3] 功能，可用 XML 格式來存取內容資料。引入 readline 是讓 Python 直譯器提供命令列歷史及指令補齊 (completion) 的功能，執行 dir() 指令後，如果有發現 'readline' 與 'rlcompleter' 兩個內建模組，即表示安裝成功。引入 libssl 是提供 SSL 加密傳輸能力，而 wv 與 xpdf 則是提供 Microsoft Word 與 Adobe PDF 檔案的索引功能。

[1]Linux rpm 操作可參考 http://www.iis.sinica.edu.tw/~marr/rpm/ 文件。
[2]如果缺少 ncurses 函式庫，執行 Python 會發現 ImportError: /lib/libreadline.so.5: undefined symbol: PC 的訊息。
[3]http://plone.org/documentation/tutorial/xml-in-plone-with-marshall

編譯時可使用管理員權限 (Root Install) 或一般使用者權限 (Root-less Install)，不論哪一種，都建議優先選擇 ZEO Cluster 方式，它能發揮多處理器的效益，適於開發環境或大型線上系統的場合，其執行的指令與參數如下：

./install.sh zeo

另一種 standalone 方式，則適於個人桌面環境的場合，其執行的指令與參數如下：

./install.sh standalone

關於編譯時的使用者權限，可考慮在開發初期 (alpha development) 採用 Root-less Install，在開發末期 (beta development) 採用 Root Install，主要差別在於 Root Install 會建立 plone 使用者帳號及目錄，這在開發初期通常是項沒必要的系統成本。

Windows 環境的安裝

下載 Plone 的 Windows Installer 檔案，選個安裝目錄，例如 C:Plone-3.0.6 資料夾，安裝程序很容易，但預設不會將 Python 安裝資訊註冊到 Windows registry 裡，想要設定 registry 的話，可參考 http://effbot.org/zone/python-register.htm 的說明。

目錄架構概況

在 Root-less Install 成功後，會在 $HOME/Plone-3.x.y 目錄裡建立 Plone 系統，採用 Root Install 則在 /opt/Plone-3.x.y 目錄裡建立系統。以 Plone 3.0.6 為例，它的第一層目錄項目如下所示：

Top-level Directory Outline for Plone 3.0.6

Plone-3.0.6
|-- Python-2.4.4
|-- bin
|-- doc
|-- lib
|-- receipts
|-- skel
`-- zeocluster

作業系統裡如果存在多個 Python 版本，複雜的相依問題可能會干擾使用者，Unified Installer 為了避免這項困擾，使用獨立自有的 Python 版本，以 Plone 3.0.6 為例，搭配的是 Python 2.4.4，如果想要新增模組 (例如 Python 對 MySQL 函式庫 3rd party package) 的支援，預設應安裝到 Python-2.4.4/lib/site-packages 目錄裡，而預設已安裝的 3rd party package 有 PIL 、 setuptools 、 openid 等。

除了 Python-2.x.y 目錄之外，其他目錄屬於 Zope 系統的架構，簡介如下：

  • bin : 內含 Zope 的工具程式，例如遺忘管理員帳號密碼時，可使用 bin/zpasswd.py 來建立緊急管理員帳號密碼。
  • doc : Zope 系統管理及開發的說明文件。
  • lib : Zope 的核心模組所在。
  • receipts : 記錄整個 Plone 系統編譯成功的時間。
  • skel : 內含 Zope instance home 的建立樣版。
  • zeocluster : Zope instance 的運作環境，包括 CMF 與 Plone 的模組。

啟動 Plone 的注意事項

  • 首次執行 bin/startcluster.sh 時，在啟動 ZEO server 後，會在 zeocluster/server/var 目錄建立 Data.fs 檔案，並在 ZODB 裡新增名為 /Plone 的 Plone site 物件，最後再啟動 ZEO client2。
  • 啟動 Plone 時會載入 po 檔，把不必要的檔案刪除，能夠節省啟動時間。主要的 po 檔位於 zeocluster/Products/PloneTranslations/i18n 目錄裡，可用 find Plone-3.0.6 -name "*.po" 找到完整檔案清單。
  • 採用 Root Install 情況時，若遇有 root restaring too frequently: quit 訊息，請檢查 zeocluster/server/var 目錄裡檔案的權限，如果 Data.fs 相關的檔案擁有人是 root，試著 chown 為 plone 後重新啟動。
  • 登入 ZMI 後，想要更改管理者的密碼，請至 /acl_users/users
  • 若有系統錯誤訊息，請優先查詢 zeocluster/client1/log/event.log 內容，其他的訊息記錄位置在 zeocluster/server/log 及 zeocluster/client2/log 目錄。

停止 Plone 的注意事項

  • 在 ZMI 裡的 Control_Panel 執行 shutdown 指令，會把 ZEO client1 停掉，但 ZEO server 與 ZEO client2 似乎繼續運行，執行 bin/shutdowncluster.sh 才會依序把 ZEO server client1 client2 都停掉。
