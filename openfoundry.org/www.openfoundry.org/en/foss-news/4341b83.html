VMware 發表 Virtual Infrastructure 3

Created at Friday, 09 June 2006 08:00      Last Updated on Thursday, 08 March 2007 08:25 Written by 謝良奇
隨著 Linux 廠商與微軟相繼要在作業系統中納入虛擬機器技術，虛擬軟體 VMware 總裁 Diane Greene 平心看待對於緊接而來的激烈競爭，但是作業系統與虛擬技術 hypervisor 之間的界面，Greene 則認為勢必要制訂出開放標準。 Virtual Infrastructure 3 發表同時，Greene 表示 VMware 不畏懼微軟或 Linux 廠商整合 hypervisor 技術到作業系統後，即將帶來的市場競爭。根據 Greene 的看法，該公司的平台遠比這些競爭者更為優異與成熟，更重要的，是 VMware 的虛擬技術與作業系統相互獨立的特點。

Novell 與紅帽預料將在今年於 Linux 作業系統內整合 Xen hypervisor 虛擬技術。目前以銷售 Virtual Server 2005 R2 產品為主的微軟，也已經宣佈計劃加速發展 Windows 伺服器作業系統原生虛擬 hypervisor 與管理平台。

雖然微軟計劃中的 hypervisor 最快也要等到兩年後才可以問世，Greene 宣稱他們不會讓微軟一貫的作業系統整合策略故技重施，因為這樣一來用戶就會被綁死在其私有的虛擬 API 或技術上。

然而，這一點也是產業觀察家最為 VMware 所擔憂的，他們擔心因為微軟將類似軟體功能整合到 Windows 內，VMware 最後將落得與其它獨立軟體供應商一樣的下場。

Greene 對此則頗有信心的表示，VMware 由於其可觀的市場領導優勢、先進的平台功能與用戶希望將虛擬技術與作業系統相互切割獨立的需求，將保持獨特的競爭優勢。

VMware 將推動硬體與 hypervisor 間的界面、檔案格式與管理界面能夠採取開放且基於標準的方式。VMware 與 Linux 社群成員正在磋商支援產品標準虛擬界面。VMware 自身的虛擬機器界面 (Virtual Machine Interface) 也是可能的標準之一。VMware 高層透露有關的協議預料會在兩個月內達成。

在與 CRN 的訪談中 Greene 虛擬管理工作小組 (Virtualization Management Task Force) 有必要支持採納標準的虛擬界面，以保障用戶投資並提供所有廠商平等的競爭地位。

Pund-IT Research 分析師 Charles King 認為，VMware 希望強調將虛擬環境與作業系統獨立看待的優點。藉由 Virtual Infrastructure 3，VMware 企圖將外界的目光拉回到虛擬環境本身，並且將虛擬環境從作業系統中獨立出來，藉此拉高其競爭地位，對於虛擬環境取得更為全面的掌握。
