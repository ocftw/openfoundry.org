JasperSoft BI 整合 ETL 軟體 推動開放源碼在地化專案

Created at Friday, 02 February 2007 08:00      Last Updated on Thursday, 05 July 2007 22:24 Written by 謝良奇

 開放源碼軟體商 JasperSoft 發表了其開放源碼商業智慧平台 JasperSoft BI Suite 的最後一位成員 JasperETL，同時積極完成全系列 BI 產品的國際化。

ETL（extract, transform, and load，擷取、轉換與裝載）的功能可以協助 BI 用戶依據企業需求進行資訊移動、整理、統整與轉換，以便確保組織資料的一致性。

JasperSoft 行銷副總裁 Nick Halsey 表示，過去幾年 JasperSoft 一直在思考如何為 BI Suite 加入 ETL 功能。該公司評估過好幾個開放源碼產品也曾考慮自行開發，直到法國開放源碼資料整合公司 Talend 的 Open Studio 引起 JasperSoft 的注意。JasperSoft 如今以 Open Studio 做為 JasperETL 的基礎，與 BI Suite 的其它成員 JasperServer、JasperAnalysis、JasperReports 與 iReport 共同整合。

Halsey 表示，JasperETL 與 BI Suite 其它產品一樣，將提供開放源碼與商用的專業版本，兩種版本都會有付費的技術支援與專業服務。開放源碼版本已經提供用戶下載，專業版本則要等到本季稍晚，專 業版本包括了共用的元資料倉儲供團隊開發、延伸套件與 JasperServer 的最佳化之用，此外也加入額外的資料擷取模組。

專長是開放源碼資料整合的獨立軟體商 Talend，其旗艦產品 Open Studio 據稱下載次數已超過三萬次。依據 JasperSoft 與 Talend 簽訂的合作案，JasperSoft 將在 BI Suite 中整合 Talend 的專業版 ETL 軟體，而 Talend 則會在歐洲透過合作伙伴與銷售管道推展 JasperSoft BI Suite。

在此同時 JasperSoft 也宣佈推出 JasperSoft BI Suite 1.2 版，具備了完整的國際化能力如 Unicode 支援、特定國家編碼支援如日本的 Shift-JIS，以及地區性資料格式等。Halsey 認為，他們擴展了其產品的功能以因應國際化需求如時區與各地的日期格式等。這些功能對於在進行計算與報表時需要考慮時間因素的全球組織相當重要。

JasperSoft 也正在致力讓其軟體的商業版本完整的在地化，當用戶登入軟體時能自動辨識用戶地點與時區。Halsey 表示第一套在地化產品是日文版，將在 2 月推出，今年稍後則會有法文、德文與義大利文版。

而為了支援其產品的國際化，JasperSoft 發起了名為 JasperBabylon 的開放源碼計劃，開發人員可共同合作進行 BI 產品的在地化，目前已有 22 個不同語言的翻譯專案正在進行，包括多數主要歐洲語言及中文、北印度文 (Hindi) 及日文。該計劃位於 JasperForge.org 專案網站上。
