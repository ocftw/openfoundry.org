古巴擁抱開放源碼軟體 Richard Stallman 現身說法

Created at Friday, 09 March 2007 08:00      Last Updated on Thursday, 05 July 2007 19:32 Written by 謝良奇
古巴正試著甩開微軟與背後象徵的私有軟體資本主義帝國，加入另一個社會主義國家委內瑞拉的腳步，改用開放源碼軟體。這兩個國家正在試圖減少對微軟 Windows 的依賴，改用由全球自由分享程式碼所發展出來的開放源碼 Linux 作業系統。 其它國家如中國、巴西與挪威也已經有類似的動作，並鼓勵 Linux 發展，微軟在作業系統上的近乎獨佔地位、私有軟體的高額成本與安全問題，可說是這股趨勢的背後主因。

一般認為開發中國家希望藉由使用開放源碼軟體，節省軟體授權上的花費，然而即使是已開發國家如日本，對於利用開放源碼軟體並減少依賴微軟，也呈現相當積極的態度。政府用戶，特別是國防單位，在敏感應用領域上對商業軟體一向懷有不信任感。

視美國為一大威脅的古巴，甚至認為改用 Linux 與國家安全相關。古巴總統卡斯楚戰友之一的通訊部長 Ramiro Valdes 在哈瓦那舉行的通訊與技術國際研討會上，就表達對於微軟可能與美國國防單位合作的不信任。他稱資訊系統為古巴對抗帝國主義的戰場。

引述微軟董事長 Bill Gates 認為包括主張揚棄私有軟體的著作權改革派是現代共產主義者的說詞，Valdes 認為，從古巴人的角度這是一項榮譽。對此微軟並未發表回應。

Bill Gates 將開放源碼社群與共產主義者連在一起的看法，誤解了選擇開放源碼軟體的關鍵因素，其實是免於受到限制，並且擁有無償更換軟體供應商的自由，允許用戶視需要修改、擴充、改善軟體的自由。

該研討會上另一位注目的焦點，是自由軟體基金會 (Free Software Foundation) 創辦人 Richard Stallman，這位堅稱著作權法違反基本道德的電腦科學家，以因為分享或修改食譜而入獄來跟著作權法做類比。他表示，私有軟體由於程式碼無法檢驗而構 成安全上的威脅，用戶無法得知開發者留下了什麼後門。他說，非公開的程式讓人無法信任。

管理某古巴大學系所的 Hector Rodriguez 認為，用不用開放源碼是一個技術主權與意識形態的問題，他所管理的系所有 1000 位學生投入開放源碼軟體的開發。Rodriguez 表示，古巴海關服務已經改用 Linux，文化部、高等教育部與通訊部等古巴內閣部會也計劃跟進。Rodriguez 所屬的系所學生正在開發基於 Gentoo 散佈套件的 Linux 作業系統 Nova，而高等教育部也有自己的 Linux 開發正在進行。

由於古巴人多半依賴政府網咖或學校等地提供的慢速網路連線，加上古巴電腦設備並不普及，儘管古巴正在訓練數以萬計的軟體與硬體工程師， Rodriguez 對於古巴全面改用 Linux 的時程則持保留態度。不過他表示 Linux 的使用率正在迅速成長。他說，2 年前古巴自由軟體社群不到 600 人，目前已突破 3000 人。
