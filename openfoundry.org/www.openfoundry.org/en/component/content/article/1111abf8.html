JBoss Rules 4.0 版推出 加入 AJAX 界面與效能提升

Created at Monday, 30 July 2007 07:46      Last Updated on Monday, 30 July 2007 09:28 Written by 謝良奇/編譯

JBoss 推出新版的開放源碼商業規則引擎 JBoss Rules，強調更好的執行效率，加上提供給非程式設計者使用的新規則開發前端界面。JBoss Rules 4.0 版這次也有了新的 Eclipse 開發者工具。

JBoss 技術產品經理 Burr Sutter 表示，4.0 版在速度上有很大的提升，處理規則耗費的時間由數分縮短至數秒。效能上的改進有賴於加入新的特殊使用案例，以及在決策表上使用新的連續模式，能將處理速度提高 30% 到 300%。此外，JBoss 表示程式碼各方面的重構也對效能有相當程度的改善。

JBoss Rules 以開放源碼專案 JBoss Drools 為基礎，用來設置企業程序中的規則，可用以部署與應用軟體相關的企業運算邏輯等功能，如定價引擎或路由節點。

4.0 版的 AJAX Web 界面讓非程式開發者也能使用 JBoss Rules。Sutter 表示，這個特點讓非程式開發者能與 JBoss Rules 互動並開發規則，讓管理階層能控管規則的生命週期。此外，JBoss 新增了對 MVEL(MVFlex Expression Language) 的支援。Sutter 表示，如今他們有了更為強大的語言環境。對於開發人員來說，也有 Eclipse 的擴充元件提供這些功能給 Eclipse 除錯者與程式編寫者使用。

JBoss Rules 可與 JBoss 企業服務匯流排 (enterprise service bus，ESB) 整合運用。規則引擎可依據訊息內容，決定要將該訊息送往何處處理。Sutter 說，ESB 用來處理訊息流，而 JBoss Rules 可視為訊息流的交通警察與 ESB 搭配使用。

JBoss 也推出了企業規則管理系統 (Business Rules Management System，BRMS)。目前JBoss Rules 仍只是一套單純的規則引擎，以技術預覽版本推出的 JBoss BRMS 可說是 JBoss Rules 邁向成熟產品的第一步。

JBoss Rules 4.0 宣稱可與 JBoss 物件導向關聯軟體 Hibernate 搭配使用，將關聯式資料庫資料轉為 Java 物件之後，企業規則將可直接從資料庫中抓取如客戶姓名等資料。
