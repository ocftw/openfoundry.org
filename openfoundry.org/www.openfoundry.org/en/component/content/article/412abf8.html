OSI 認可 CDDL 授權 Sun 釋出部份 Solaris 程式碼

Created at Tuesday, 18 January 2005 08:00      Last Updated on Thursday, 08 March 2007 08:03 Written by 謝良奇
歷經多月來的外界猜測與疑慮，Sun 終於在日前踏出開放 Solaris 源碼的第一步，將 Solaris DTrace 技術以新近由開放源碼協會（Open Source Initiative，OSI）認可的通用開發與散佈授權（Common Development and Distribution License，CDDL）釋出。預計於今年 第二季，就可以見到完整的 OpenSolaris 散佈套件釋出。 似乎要一掃過去業界疑慮，Sun 執行長 Scott McNealy 在媒體前力陳 Sun 與開放源碼長久以來的淵源，宣稱 Sun 從創始以來，一直堅信開 放標準與開放源碼流程的價值。McNealy 表示藉由 Open Office、 Gnome 及 NFS，Sun 目前是開放源碼最大貢獻者之一，加上針對 Open- Solaris，Sun 所釋出的 1600 種以上相關專利，足以令其它廠商的貢 獻失色。他表示，這對 Sun、社群、開發者與客戶而言，都可說是重大 里程碑。

CDDL 是 Sun 以 Mozilla Public License 為基礎的授權方案。根據 Sun 執行副總裁 John Loiacono 表示，選擇 CDDL 有三大理由，利於 程式碼重用、授權變更不會影響使用原授權之開發者，以及該授權要 求程式碼修改必須公開。除了 Solaris 外，有消息指出 Sun 正考慮以 CDDL 釋出 Java Enterprise System（JES）。由於 JES 是用來建構 Java 應用軟體與服務的基礎平台，此一動作對 Sun 的合作夥伴、客戶 及開放源碼社群將有重大影響。

Linux 與開放源碼方案供應商 Cybersource 執行長 Con Zymaris 認 為，開放源碼開發如今擁有遠大於 Java 的動能。Sun 可藉由將 Java 開放源碼或同意外界發展開放源碼版本，重新為 Java 平台注入活力。

Robert Frances 分析師 Stacey Quandt 則認為 JES 是 Sun 的獲利 來源， Sun 現在應該沒理由開放 JES 源碼。Sun 營運長 Jonathan Schwartz 有不同看法。他認為這完全忽視了軟體產業的基本變革， 這就好像說 Google 不應該免費服務，而且賺不到錢是一樣的。 Schwartz 表示，對 Sun 而言，開放源碼是資本主義也是絕佳的商業 契機。

Quandt 表示，為了在開放源碼競爭中勝出，Sun 得讓其客戶看到 Solaris 10 是邁向透明化、標準以及價格效能比優越的產品。Quandt 認為，CDDL 只是第一步，Sun 有機會透過開放源碼的 Solaris，在 開放源碼社群中取得領導地位。而 OSI 認可 CDDL 的動作，使得 Sun 能在 Linux 與開放源碼之間做一個清楚的劃分。
