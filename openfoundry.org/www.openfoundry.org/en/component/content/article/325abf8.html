Linspire 推出 Freespire 套件 內含已授權私有軟體

Created at Friday, 28 April 2006 08:00      Last Updated on Friday, 09 March 2007 01:26 Written by 謝良奇
桌面 Linux 系統供應商 Linspire 在第四屆年度桌面 Linux 大會 (Annual Desktop Linux Summit) 公佈了新的 Linux 散佈套件 Freespire。在許多開放源碼擁護者的眼裡，這套由 Linspire 贊助的免付費 Linux 散佈套件，或許不能被視為真正的開放源碼軟體，因為 Linspire 將會在這套散佈套件中，整合獲得授權的私有軟體。

Linspire 總裁兼任執行長 Kevin Carmony 在該會的演說中公開了 Freespire。以 Debian 為基礎的 Freespire，雖然與商業版本的 Linspire 散佈套件關係密切，但卻是由社群提供支援的專案。其運作方式與 Red Hat Linux 的 Fedora Core、SUSE Linux 的 openSUSE 如出一轍。 Freespire 和上述專案的不同點在於 Linspire 將會在該散佈套件內整合取得授權的私有軟體及驅動程式，用戶就可以在該系統上使用合法軟體觀賞 DVD。 將整合到 Freespire 散佈套件的私有軟體包括 codecs、設備驅動程式以及第三方字型。Linspire 表示將持續發展自家的 Linspire OS，他們也會利用 Freespire 專案來協助改善商業版的 Linspire OS。 Carmony 也明白，納入私有軟體可能因為違反了純粹開放源碼原則，會讓 Linux 社群裡部份人反感，但同時這些私有軟體與 codecs，會幫他們甚至是整個 Linux 社群，吸引更多一般用戶與開發人員。顧慮到這一點，Freespire 將會有兩套版本，一個是未含私有軟體的全開放源碼版本，一個則是有已授權私有軟體的版本。 Freespire 的實際釋出時間要等到 8 月後，下一次的 LinuxWorld West 大會是可能時間點。Carmony 表示延遲釋出是為了讓用戶跟開發人員有機會表達對 Freespire 專案的看法。該公司也急切希望獲得來自外界的回應。 雖然 Linspire 一向以發展高親和力的桌面 Linux 自居，然而這次決定在 Freespire 納入私有軟體，並非全為了一般用戶。Carmony 表示，他們希望新增的功能可以讓 Linux 社群外的開發人員注意到 Linux。他說 Windows 跟 Mac 的開發者多半不喜歡 Linux，希望這次能一改他們的刻板印象。 另外，Carmony 也提到同時提供開放源碼版本與私有軟體版本，可以藉此刺激開放源碼軟體的發展。他說，假如同時提供用戶與開發者 NVIDIA 版本與開放源碼版本的驅動程式，一旦他們發現私有版本的效能比較好，將有助鼓勵開放源碼版本的發展團隊加緊趕上。
