Samba 專案將改採 GPLv3 授權 衝擊微軟與專利協議合作對象

Created at Tuesday, 10 July 2007 16:00      Written by 謝良奇

開放源碼檔案與列印分享專案 Samba 的開發群們公開宣佈，Samba 專案將採用剛釋出的GNU 通用公眾授權第三版（GPLv3）。Samba 成為第一個採用該授權的主要專案，如此一來對簽訂軟體專利契約的廠商將產生重大影響。

Samba 首席開發人員 Jeremy Allison 發表 Samba 將改用 GPLv3 的消息。他說，Samba團隊經過內部討論後，決定採用 GPLv3 與 LGPLv3 授權。Samba 預計下一波將釋出的新版本 3.0.26，也為此改版本號為 3.2.0。3.2 或之後的版本就是 GPLv3，3.0.x 或之前的版本則維持 GPLv2。

Samba 是讓非 Windows 作業系統與 Windows 連接的網路軟體，採用的是 Windows 網路協定 SMB/CIFS 的逆向工程版本。Linux 與其它基於 Unix 的作業系統，如 Mac OS X，都是依賴這套軟體與 Windows 環境相連接。

Samba 採用 GPLv3 影響之大，主要和此專案在普及程度有關，其影響所及，特別是微軟，以及與微軟簽訂軟體專利協議的公司。因為根據自由軟體基金會（Free Software Foundation）的看法，一旦 Novell 將 GPLv3 授權的程式碼納入散佈套件中，微軟允諾提供給 Novell SUSE 使用者的專利保障，也將自動付予給該程式碼所有用戶。

微軟日前已聲明該公司不受 GPLv3 條款所限，即便 Novell 的 SUSE Linux 企業版包含了 GPLv3 程式碼。微軟目前正是該版本的訂購認證的供應者。微軟表示其行動並無可錯解為接受做為 GPLv3 簽署之一方，也不應假定需負擔該授權下之任何法律責任。

此外，對於跟微軟簽訂軟體專利協議的公司，Samba 採用 GPLv3 也是一個壞消息。Allison表示，2007 年 3 月 28 日之後簽訂的專利契約，只要在 GPLv3 第 11 章被視為進行差別對待，就明顯違反了 GPLv3。任何在上述日期後簽了這類專利契約的 Samba 散佈商，將無權再散佈採用 GPLv3 授權的 Samba 版本。

這一來 Xandros、Linspire 這兩家跟隨 Novell 之後與微軟簽訂合作協議的公司，將受到影響。雖然微軟聲稱其專利協議中的合作對象不受 GPLv3 條款限制，產業觀察家卻不同意微軟的看法。

同時，Samba 授權更改也會影響到使用 Samba 程式庫的公司。因為 Samba 也會在程式庫上採用 LGPLv3，假如連接到 Samba 程式庫的軟體採用的是僅允許 GPLv2 的授權，就會發生不相容的問題。Allison 認為，如果希望使用 LGPLv3 的程式庫，那麼就得修改軟體授權。像是採用 GPLv2 或以後版本的授權宣告，就可以連接至 LGPLv3 授權。

Samba 團隊會持續對 Samba 的 GPLv2 版本提供安全更新，但只會為 GPLv3 版本開發新功能。Allison 也表示 GPLv3 的反數位權利管理（Digital Rights Management，DRM）條款應該不會影響到 Samba 用戶。
