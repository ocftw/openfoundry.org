Oracle Unbreakable Linux 支援計畫成果可觀

Created at Tuesday, 20 November 2007 08:00      Written by 謝良奇/編譯

Oracle 宣佈該公司一年前公開的 Unbreakable Linux 支援計畫，已經有 1500 個企業用戶加入。同時，該計畫在資格認證與 Oracle 驗證架構計畫（Oracle Validated Configurations Program）上也吸引了相當多的合作夥伴參與。

Oracle Unbreakable Linux 支援計畫由於針對 Linux 與全系列 Oracle 軟體堆疊，提供單一的企業級品質服務窗口，已經成為 1500 家企業用戶的選擇，並持續成長中。其中超過 1450 家企業用戶是在最近 6 個月中簽訂該服務，Oracle 執行長 Larry Ellison 在 OpenWorld 使用者大會上自豪表示，這個數字一點都沒有灌水。用戶數比起 6 個月前 Oracle 宣佈的 26 個用戶，有相當顯著的成長。


Unbreakable Linux 計畫針對紅帽企業版 Linux（Red Hat Enterprise Linux，RHEL），以及出自該作業系統的 Oracle 散佈套件，還有這些作業系統上的應用軟體，提供商業支援。價格僅為紅帽服務價格的一半，每年 99 至 2000 美元不等。

Oracle 的 Oracle 企業架構長（Oracle Chief Corporate Architect） Edward Screven 表示，Oracle Unbreakable Linux 支援計畫廣受好評已超出預期，顯示 Linux 使用者尋求的是，像 Oracle 提供給資料庫用戶的同等級高品質支援。Oracle 會持續肩負起滿足 Linux 用戶需求的使命，擴展已相當成熟的合作圈，更重要的是，會持續技術上的貢獻，務求使 Linux 成為適於任務關鍵應用軟體的作業系統。

相對於 Oracle 的高成長，紅帽仍是企業 Linux 市場的領導廠商。紅帽執行長 Matthew Szulik 在 6 月時就曾表示，該公司單就 2008 會計年度的第一季，就增加了數千位客戶。該公司表示，最近一季其簽訂服務營收包括支援合約在內，為 1 億 920 萬，年成長為 29%。前 25 大支援客戶，皆以前期合約的 122% 價格續訂合約。

雖然紅帽的 Linux 業務仍遠勝於 Oracle，從 OpenWorld 上不少用戶表達對紅帽支援的不滿看來，紅帽並非穩居市場領導地位。

例如，Stuart Maue 的資訊長 Brad Maue 表示，Oracle 的支援只有紅帽的一半價格，但他們卻感受的待遇卻多了兩倍。這一家法律稽核公司約在 1 年前將 Linux 支援服務從紅帽轉給 Oracle。

Yahoo 的資深資料庫工程師 Tim Heath 表示，由於與紅帽合作關係中某些不好的經驗，該公司開始使用 Oracle 的 Linux 支援。他說，紅帽在推出 RHEL 5 後，有些 Linux 的臭蟲，Yahoo 一直沒辦法促使紅帽解決。Yahoo 目前有 5 萬部使用 RHEL 4 的伺服器。簽訂 Oracle 的支援服務後，Yahoo 同時也持續使用紅帽的支援服務。Heath 表示這麼做希望能在兩者間製造競爭感，看誰能先把臭蟲修掉。

來自 Oracle 夥伴網路（Oracle PartnerNetwork）會員，通過 Oracle 企業 Linux（Oracle Enterprise Linux）認證的解決方案，向用戶確保其環境經過完整測試並可立即運作，是 Oracle Unbreakable Linux 支援計畫的最佳後盾之ㄧ。

除此之外，Oracle 驗證架構計畫中，多達十數家基礎架構廠商與 Oracle 合作，提供預經測試、通過驗證且提供支援的架構，其中包括軟硬體、儲存與網路元件與最佳實務經驗，都進一步簡化了 Oracle 解決方案在 Linux 上的部署工作。透過這個計畫，Oracle 與 AMD、Cisco、Dell、HP、IBM、Intel 等公司合作，設計與測試 Linux 上的全系列堆疊。

為進一步顯示協助企業成功部署 Linux 的決心，Oracle 已經向 Linux 社群貢獻相當可觀的程式碼，包括支援極大規模的新檔案系統 btrfs，藉由 Oracle 叢集檔案系統（Oracle Cluster File System）持續推動叢集技術發展，發起開放源碼介面專案公開 T10 資料整合欄位標準（T10 Data Integrity Field），以協助縮短系統停機時間等等。

紅帽官方試圖要淡化 Oracle 的 Linux 計畫的殺傷力。該公司技術長 Brian Stevens 表示，充其量，Oracle 只是增加旗下應用軟體在 RHEL/Unbreakable Linux 上的執行速度，並減少臭蟲。這與服務好壞無關，單純只是因為那些 IT 堆疊（IT stack）掌握在 Ellison 的手中。
