回顧 2003—Linux 最好與最壞的一年

Created at Wednesday, 31 December 2003 08:00      Last Updated on Friday, 09 March 2007 00:15 Written by 謝良奇
2003 告終，隨著我們的腳步即將踏入 2004，又該是對過去一年做番省思、回顧，順便展望展望未來的時機。對於變化迅速的開放源碼軟體界，一年不可謂不長。在 2003 年的開放源碼圈子裡，有訴訟、有重要的軟體發行、廠商們在市場上來來回回，從中看見開放源碼的興起與蓬勃，固然令人喜悅，然而，這一年也有不少紛擾。 2003 開放源碼軟體界，縱觀之下，最閃亮的明星，非 Linux 莫屬。這一年裡，Linux 不僅在桌面、伺服器應用，各有不少收穫與成長，Linux 本身更因一場訴訟，成了開放源碼軟體，最引人注目的箭靶。

無疑的，Linux 今年在桌面使用上，有特別搶眼的表現。Linux 在桌面上的重要進展，多半是來自於辦公室軟體與企業應用軟體的推動。KDE、Gnome、OpenOffice 等軟體越見成熟，帶動 Linux 在桌面市場的進展。Linux 被正視為 Windows 的潛在對手。

在伺服器方面，Linux 的表現亦不俗。伺服器作業市場在 2003 年裡有不少的變動。Linux 獲得許多傳統 Unix 顧客，以及微軟 Windows 使用者的青睞。IBM、Sun 和 Dell 將 Linux 做為解決方案之一，搭載於硬體上銷售。在網格運算（grid computing）和描繪場（render farms）上，Linux 逐漸成為普遍方案之一。例如電影辛巴達（Sinbad），便是採用 Linux 在 HP 硬體上製作而成。

2003 年裡，不管是從技術，或是企業應用的角度看來，Linux 都可說是邁進相當大的一步。在低階應用上，有賴於 Linux 部建與維護成本低廉，Linux 已成為中小企業檔案及網路伺服器的選擇。

經過長期的等待，眾所矚目的 Linux 2.6 核心終於在 2003 年 12 月 18 日釋出，除了支援 64 位元的運算外，還有許多新功能與改進。過去一年裡，Samba、Apache 以及不少重量級的開放源碼軟體，都有相當不錯的進展。2.6 核心推出後，相信 Linux 在核心企業商務應用上將更上層樓。

支持 Linux 的社群或個人，恐怕會對 2003 一整年有著又愛又恨的心態。因為雖然 Linux 有著相當的成長，但在逐步蓬勃的同時，這套開放源碼作業系統卻成了商業戰火下的箭靶。2003 可說是 Linux 最好與最壞的一年。

今年初，SCO 針對 IBM 提出的訴訟案，憾動了整個社群，很快地，整件訴訟擴展成對於 Linux 合法性基礎的質疑與挑戰。最終，SCO 將矛頭指向了採用 Linux 的大型企業。2003 年 5 月，SCO 向全球 1500 家大型企業發出律師信，使得企業使用 Linux，蒙上一片法律責任的陰影。8 月時，SCO 宣稱將對所有在雙處理器上使用 Linux 的公司，收取 1399 元美金的權利金。該公司在 11 月宣佈將控告採用 Linux 的大型企業，而這是該公司首次以侵害智產權，採取法律行動。

儘管如此，來自 SCO Group 的威脅，雖然造成社群的震撼，但卻未能降溫各方對於 Linux 的擁護。

2 月時舉辦的桌面 Linux 大會（Desktop Linux Consortium），展現開放源碼市場認真建構 Windows 桌面系統競爭對手的企圖。許多分析師仍對從 Windows 轉移至 Linux 的成本效益表示懷疑，但這一點並未減緩各方投入此領域的腳步，這些努力包括 Red Hat 的 Linux Technical Workstation、Sun 的 Java Desktop System，以及 Novell 收購桌上 Linux 供應商 Ximian 與 SuSE 的動作。

目前 Linux 桌面應用的斬獲，多半在企業與政府單位中。有名的例子包括德國慕尼黑市政府將 14000 部電腦由 Windows 轉移至 Linux。

過去 Linux 企業 IT 支援不受重視的情況早已不再，IBM 是最好的一個例子，然而，Novell 於 11 月在 IBM 支援下，以 2.1 億美元收購 SuSE Linux，加上資料庫龍頭 Oracle 從桌面到伺服系統對於 Linux 的全面支援，將 Linux 轉變為企業資訊部門採購的安全選擇。這些企業採構者重視廠商穩定性，更甚於技術創新性。

因此 Linux 在 2003 年裡，就像基於 Intel 的伺服器和 Windows 桌面電腦般，成為企業應用環境中的常客。
