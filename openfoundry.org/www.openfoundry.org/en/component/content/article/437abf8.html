Wind River 公開 Eclipse 設備軟體開發專案

Created at Tuesday, 17 May 2005 08:00      Last Updated on Thursday, 08 March 2007 08:28 Written by 謝良奇

特別針對電信級 Linux（Carrier Grade Linux）與核心除錯乃至應 用軟體開發各項 Linux 開發需要，Wind River 日前擴展旗下 Linux 解決方案對 Linux 開發週期各階段開發者需求的支援。

該公司解決方案將供應設備製造商整合 Linux、中介軟體與開發套件 Wind River Workbench 的完整執行時期環境。此外，該公司也延續 對開放源碼專案的支持承諾，公開基於 Eclipse 的設備軟體開發 （device software development）上層專案。

Wind River 產品規劃與管理副總裁 John Fanelli 表示，這項伴隨 年初 Linux 釋出與該公司 Linux 平台廣受市場接受的動作，可為其 他 Linux 供應商鋪路。Fanelli 指出，該公司客戶需要該公司超過 二十年支援與專業服務經驗背書的 Linux 解決方案，整合最新的 kernel.org 釋出。

為了應付更複雜軟體的支援需求，設備製造商必須在兼顧品質與可靠 性同時縮短複雜產品的交付時期。也因此，製造商選擇採納有助創新、 彈性與成本控制的設備軟體最佳化（Device Software Optimization， DSO）策略。Wind River Linux 解決方案有助客戶交付更具連接性、 安全性、可靠性與品質的 Linux設備軟體。

OSDL 開放源碼架構專家 Bill Weinberg 表示，Wind River 在對開 放源碼的支援與提供設備軟體領域額外選擇上，有相當的進展。該公 司提供了廣泛的解決方案，並支援多種作業系統與 Eclipse 工具， 在 Linux 設備軟體上扮演關鍵投入角色，並積極參與 OSDL 的電信 工作小組（Carrier Grade Working Group）。

該公司基於 kernel.org 的 Wind River Linux 平台包括 General Purpose 平台及 Network Equipment 平台。該公司行銷副總裁 John Bruggeman 表示，General Purpose 平台與 Network Equipment 相 似，但未包括叢集與高效（high-availability）能力。該平台基於 2.6.10 核心，支援電信級 Linux 版本 2（Carrier Grade Linux version 2）、ATCA 及 Compact PCI、Workbench 2.3 開發套件支援 等。該平台將提供先期試用版本給特定客戶，並預定於今年六月發表。

除了以上平台外，根據 Bruggeman 表示，Wind River 正在發展軍事 與航太市場的 Linux 平台，但相關細節則未透露。

Wind River 同時也發表了基於 Eclipse 3.0 版本的開發套件 Work- bench 2.3，此版本支援了 Wind River Linux、Red Hat Linux、 kernel.org Linux 及 MontaVista Linux，並首次加入該公司合作夥 伴 Express Logic 的 RTOS Express Logic 支援。

此外，延續該公司日前宣佈將擴大針對 CELF、Eclipse 在內等多項 開放源碼專案支援的承諾，Wind River 此次即提出一項基於 Eclipse 的上層專案，設備軟體開發平台專案（Device Software Development Platform）。Bruggeman 表示，該專案已為社群所接受，並開始穩健 運作。
