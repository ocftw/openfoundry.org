Apache Synapse 提供企業網路服務基礎架構

Created at Tuesday, 23 August 2005 08:00      Last Updated on Friday, 09 March 2007 01:55 Written by 謝良奇
Apache 軟體基金會（Apache Software Foundation）不久前推出了 發展應用程式整合的開放源碼專案 Synapse，該專案結合其他開放源 碼軟體將可提供完整網路服務（Web service）基礎架構，在服務導向架構（service-oriented architectures）各家廠商商業產品外，提供用戶替代性之開放源碼產品。

Apache Synapse 專案將提供企業服務匯流排（enterprise service bus，ESB）應該具備的多項功能，企業服務匯流排透過 XML、網路服 務界面與標準化規則路由，可在企業系統間提供安全的交互運作機制。 日前由 Covalent Technologies 等公司合資發起的 WSO2 是 Synapse 專案的發起者，其目標在於提供 Apache 網路服務開放源碼軟體相關 的專業服務。除了 WSO2 外，Synapse 的提交者（committers）還包括企業服務匯流排廠商 Iona、Sonic，網路服務管理供應商 Infravio、 Blue Titan，該專案初期程式碼由 Sonic 與 Infravio 提供。 Synapse 並非為了要取代具完整功能的商業企業服務匯流排產品，然而，一旦結合其他開放源碼套件，如 Apache Agila、Apache Strut、 Apache Axis 等，將可提供企業用戶與開發者商業網路服務整合工具外的低階、開放源碼替代選擇。 Sonic 副總裁兼任首席技術推動者 Dave Chappell 表示，藉由共同 發起此一開放源碼專案，該公司希望建立產業標準並推動網路服務成為應用軟體間的中介機制。Chappell 也提到，Synapse 的網路服務 中介能力不會與其他商業整合軟體直接競爭，而是提供可供應用於企業服務匯流排或網路服務管理軟體中的架構性元件。 市場研究公司 Ovum 分析師 Ian Charlesworth 認為，開放源碼的契機在於切入商業整合架構的基礎部份，提供市場可資運用的額外選擇， Apache Synapse 即是經典案例。 儘管商業公司參與社群專案並非首例，去年宣佈計畫整合相關企業服 務匯流排與服務導向架構專案的 ObjectWeb 協會卻表達了對於商業 公司主導開放源碼社群的憂慮。對此，Apache 基金會董事 Jim Jagielski 表示 Apache 長久以來就鼓勵商業與非商業共同參與專案， 但這並不意味著該基金會放棄專案主導權。
