亞洲開發者調查：70% 開發者曾採用過開放源碼模組

Created at Friday, 15 December 2006 08:00      Last Updated on Monday, 05 March 2007 19:16 Written by 謝良奇

Evans Data Corporation 近期所做的一份調查報告指出，開放源碼在亞洲地區的使用率，近年來有大幅提升的現象，如今超過 70% 的亞洲軟體開發者都在應用軟體開發中，採用過開放源碼模組。


Evans Data 的這份報告顯示，亞洲地區使用開放源碼的開發人員，近三年來有超過 40% 的成長。70% 的開發者表示在應用軟體中採用過開放源碼模組，19% 的開發者表示不曾使用，10% 的開發人員則不知道開放源碼。Evans Data 在 2003 年的類似調查中，有 50% 的開發者曾使用過開放源碼。

Evans Data 這次的調查對象包括中國、印度、日本、澳洲與紐西蘭，遍佈亞洲地區共 396 位軟體開發者。

該研究機構表示，開放源碼的普遍化，為面臨解決複雜問題的開發者省去許多功夫。因為開發者可以直接閱讀應用軟體的程式碼，了解軟體如何運作與修改。這一點與彷彿在黑盒子中執行的私有軟體完全不同。

Evans Data 研究人員表示，透過開放源碼，開發者可以藉用前人的心血，免去解決重覆問題之苦。想知道資料庫連接軟體如何運作，多得是範例程式，甚至可以合法地使用在應用軟體中。

Linux 則特別受到亞洲硬體製造商的青睞，原因不外乎 Linux 提供了低成本軟體測試的平台。台灣軟體開發商訊連科技 (Cyberlink) 最近正與仁寶電腦 (Compal) 合作採用 Linux 的 TV Phone，該專案成員 Renee Chen 表示，Linux 的主要優勢之一，就是成本低於 Windows 解決方案。手機製造商 Datang Mobile 副總經理 Liu JiTang 指出，所有競爭者都在致力讓 Linux 成為行動裝置的關鍵元件，他們也正在積極參與中。

Evans Data 針對北美開發者所做的調查報告則顯示，Visual Basic 系列程式語言使用率有大幅滑落的跡象，減少了 35%。Java 則穩坐領先地位，有 45% 的使用率。Evans Data 總裁 John Andrews 認為，微軟從 90 年代初期一直穩佔程式語言的領導地位，如今情況已不同。描述語言和 Java 等已經減低了 VB 未來的市場潛力。

針對北美超過 430 位開發者所做的這份調查顯示，超過 70% 的開發者表示未來一年內會從事 Web 相關開發。80% 的開發者正在撰寫互動型網路應用軟體 (Rich Internet Applications)，Ajax 的使用率成長了 10%，上升到 28%。
