Alfresco 虛擬檔案系統 JLAN 採用 GPL 釋出

Created at Tuesday, 23 October 2007 16:00      Last Updated on Friday, 02 May 2008 22:04 Written by 謝良奇
開放源碼企業內容管理廠商 Alfresco Software 宣佈 Alfresco JLAN 以 GNU 公眾授權（GPL）方式成為開放源碼產品。過去 JLAN 已經納入 Alfresco 開放源碼內容管理解決方案中，如今開發夥伴可以在產品中，提供 JLAN 客制化功能。之前該公司早已將該開放源碼企業內容管理軟體的其他部份，同樣以 GPL 授權釋出。

之前名為 Alfresco 智慧型檔案系統（Alfresco Intelligent File System，AIFS），JLAN 是個嵌入式虛擬檔案系統，對使用者而言其使用方式就有如共享磁碟一般。JLAN 利用純 Java 的微軟 Window 通用網際網路檔案系統（Common Internet File System，CIFS）實作，讓內容、系統管理資訊、資料庫欄位看起就像共享磁碟。由於共享磁碟是今日大多數電腦使用者實際使用的文件管理介面，開發者藉著 JLAN 便可在內容系統上，提供智慧型的共享磁碟介面。

純 Java 的 CIFS、NFS 與 FTP 伺服器和用戶端實作，比擬原生檔案系統的高效能，提供 NTLM、NTLMSSP、SPNEGO、Kerberos/AD 企業認證功能，即時存取與離線存取，以上都是 JLAN 引以為傲的功能。

JLAN 之前由 Alfresco 整合到 Alfresco 企業內容管理伺服器（Alfresco Enterprise Content Management Server）中，作為 OEM 的獨立可嵌入選擇元件，讓合作夥伴迅速地將 JLAN 的功能加入自己的產品中。Alfresco JLAN 的客戶包括了 Avaki EII、Caringo、EMC Documentum 與 Veritas。這些公司在內容管理系統、資料庫以及客戶關係管理系統等內容推動的應用軟體上，將 Alfresco JLAN 作為共享磁碟介面之用。

此外，以 GPL 釋出 JLAN，此舉對於像 SpikeSource 這類的 Alfresco 平台開發者，有助其在核心內容上整合客制化的 JLAN 技術，相信是一大利多。

Samba 團隊成員也是 jCIFS 專案共同發起人 Chris Hertel 表示樂於見到 Alfresco 此次對開放源碼做出的貢獻。各開放源碼 CIFS 實作都將因該社群的經驗，以及對該協定的運用而受惠。

Alfresco Software 技術長 John Newton 表示，他們因 Alfresco JLAN 擴展了該公司 OEM 的潛能，提供簡易介面與低成本開放源碼解決方案，可輕易嵌入各種系統中。包括全球最大軟體公司在內，超過 50 個客戶已經在他們的產品中納入 Alfresco JLAN。

Alfresco 是企業內容管理領域主要的開放源碼替代方案，也是第一套可嵌入的企業內容管理系統，下載次數已超過 70 萬次，擁有 2 萬 5 千個應用網站與 300 個客戶，其中大多是全球 2000 大企業。

在 GPL 授權外，JLAN 提供自由與開放源碼軟體例外條款（FLOSS Exception），Alfresco 的自由與開放源碼軟體例外條款，允許使用了 OSI 認可開放源碼授權的開放源碼軟體，無須以 GPL 授權整個軟體套件，即可納入 Alfresco 社群軟體。為不希望採用 GPL 的開放源碼軟體，提供了更多選擇。此舉也有助吸引其他開放源碼內容管理系統如 Drupal、Joomla!、Magnolia 等，採用 JLAN 並加以改進。
