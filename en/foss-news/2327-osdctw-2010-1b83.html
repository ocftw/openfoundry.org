OSDC.TW 2010 圓滿落幕，我們明年見！

Created at Wednesday, 05 May 2010 08:00      Last Updated on Tuesday, 11 January 2011 21:58 Written by 簡信昌

台灣 open source 開發者的重要盛會，OSDC.TW (Open Source Developers Conference, Taiwan) 今年在 4/24-25 假中央研究院人文科學研究所國際會議廳舉辦，今年參加的人數又突破往年新高，總計有超過 400 人參與。兩天的議程一共有超過 32 個主題，以及超過 15 位的國外講者，精采程度超越過往四年的 OSDC.TW。

今年最重要的來賓是 Javascript 教父級的人物 Douglas Croakford，目前任職於 Yahoo! 的 Javascript 架構師，在最近幾年網路技術大幅躍升之際，Javascript 的重要性更是不可同日而語，但是就在這重要的時刻，未來的 Javascript 方向為何，以及另一個大家關注的 HTML5 的發展為何？此時 Douglas 提出了一個關鍵的重點，也就是安全性的問題。當我們對於網路的依賴日益增加的同時，安全性的考量卻似乎沒有同時被注意到。因此 HTML5 以複雜的規格準備讓網路發揮更大的影響力時，要怎麼能同時兼顧安全性是一個重要的議題。

此外，雲端技術也是近幾年熱門的話題，open source 開發者社群在這一部分也沒有缺席。特別是目前重要的雲端平台，Hadoop 以及 Hbase 都是由 Apache 基金會所贊助的專案，而這次 OSDC.TW 邀請到趨勢科技的架構師也是 Hbase 核心開發者－－Andrew Purtell，前來討論關於 Hbase 接下來開發的重點，尤其對於台灣的政府單位以及產業界大聲疾呼要積極推展雲端產業，相信 Andrew 對於 Hbase 發展的策略與方向的討論，能夠對相關的產業人士有一些提醒的作用。

另一個跟雲端有關的議題則是網路與雲端的結合運用，也就是讓 Ajax 運用雲端的相關技術及結合，這也是這一年來比較有趣的運用。而在行動裝置上，Android 當然是開放源碼的當紅炸子雞，台灣也因為手機產業的發達，有不少開發者投入到這個領域。其中的 0xlab 更是相關領域的翹楚，0xlab 的代表性人物 jserv，今年也在 OSDC.TW 跟大家分享了他們在相關方面的新成果。另外一個則是少見的主題，用 Scala 開發 Android 的應用程式，雖然 Scala 在台灣的使用者應該算是相當少數，但卻因為 Android 的當紅，讓這場分享的聽眾也相當踴躍。

提到 Android，就不能不提到 Google，而 Google 除了 Android 之外，去年另一個讓人好奇的開放源碼專案則是 GO。這是一個新的程式語言計畫，當然緣起於過去的 Plan9，他是介於像 C 那樣低階，與 Python 這麼高階的中間，讓程式人員有比較好的控制，卻又不需要自己管理相關的資源。

今年的 OSDC.TW 講者幾乎是創下歷年來國外講者最多的紀錄，除了表示 OSDC.TW 慢慢達到讓台灣開放源碼的開發者能跟國外的開發者交流的目的之外，也表示 OSDC.TW 受到了許多國外開發者的肯定。希望透過 OSDC.TW ，讓台灣開放源碼的開發者能吸取最新的技術資訊，並與國外的開發者進行面對面的討論與技術交流。

筆者特別在此感謝自由軟體鑄造場（OSSF）協助本屆 OSDC.TW 2010 的場地租借事宜，研討會場地一直是主辦 OSDC.TW 最大的問題之一。而此次租用的中研院人文所國際會議廳有口皆碑，不僅與會者感到舒適，連許多國外講者也都讚譽有加。能夠借用到這樣國際等級的會議廳，確實為 OSDC.TW 2010 的順利舉辦打下了成功的基礎。
