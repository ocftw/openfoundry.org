Motorola 創造開放互通行動 Java 開發平台

Created at Tuesday, 16 May 2006 08:00      Last Updated on Friday, 09 March 2007 01:17 Written by 謝良奇
Motorola 為了讓行動產業在 Java 技術上有更密切的整合，日前宣 佈成立開放源碼計劃 https://opensource.motorola.com/，將公開自 家的 Java 測試框架與範例測試案例源碼。該計劃目標在於透過開放 源碼專案，為行動 Java 平台創造出更為一致而開放的環境。

結合 Motorola 的 MOTODEV 開發人員支援計劃，https://opensource.motorola.com/網站藉著架構出該公司與 Java 社群間開放而簡單的 溝通管道，開發者將更容易取得世界級程式碼、應用軟體與智財資本 (IP)。開放源碼提供程式碼的方式不但可以加速 Motorola 平台上的 應用軟體開發，更能有助創新與新行動服務的普及。

https://opensource.motorola.com/ 將提供 Motorola Linux 裝置如 A1200/MING 與 A780/E680 的核心與驅動程式原始碼。除了測試框架 與範例測試案例等 Java 技術外，Motorola 不久後也會提供如 MIDP 3.0 等 Motorola 主導的 JSR 相關程式碼、文件與規格。

藉由這一連串開放源碼的動作，Motorola 希望為行動 Java 平台創 造更為開放而一致的發展環境，真正達到一次開發處處可用 (writeonce, run everywhere) 的目標。Motorola 行動設備組織副總裁 Cherlyn Chin 就表示，他們的目的解決相互運作性 (interoperability) 的種種問題，同時培育出更好的發展環境。

Motorola 行動裝置軟體資深主任與首席架構師 Mark VandenBrink 認為，Motorola 的計劃是讓開發者無須授權就得以使用 MIDP3。他 說，開放源碼社群的基礎是資訊與意見的交流，Motorola 希望透過 公開程式碼，加速這類的資訊交流，並為開放的行動開發環境投注一 份心力。

Mark 表示，對於行動產業而言，為使用者創造更為獨特的使用經驗 是至為關鍵的，Java 可說是達成此目標相當重要的技術。目前行動 裝置商都要取得其它公司的 JSR 授權後自行修改，造成平台的不一 致性。Motorola 認為把這些 Java 面臨的不一致障礙掃除，將有助 改善無線 Java 開發生態，隨之提高 Java 的相容性、親和性與易用 性。
