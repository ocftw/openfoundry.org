甲骨文為 Linux 叢集提供解決方案 Oracle Clusterware

Created at Tuesday, 01 April 2008 08:00      Written by 謝良奇/編譯
甲骨文 (Oracle) 繼 2006 年貢獻 Oracle Cluster File System Version 2 給 Linux 核心，再度提供叢集解決方案。甲骨文宣佈 Basic 與 Premier 的 Oracle Unbreakable Linux 支援用戶，可以下載並部署 Oracle Clusterware，且無須額外費用。

甲骨文將 Oracle RAC (Real Application Clusters) 套件的 Linux 叢集元件獨立出來，提供給 Unbreakable Linux 支援計畫的用戶。獨立的 Oracle Clusterware 軟體，支援 Red Hat Enterprise Linux 與 Oracle Enterprise Linux。 作為 Oracle RAC 的基礎元件，Oracle Clusterware 可獨立運作，有助於確保甲骨文或協力廠商應用軟體的運作。甲骨文開放源碼產品行銷資深主任 Monica Kumar 表示，藉由Clusterware，用戶可開發出提供高可用性 (availability) 與延展性 (scalability) 的環境。

Oracle Clusterware 為可移植的叢集軟體，可將個別伺服器協調結合在一起，有如單一系統般運作。Oracle Clusterware 因此為叢集環境下的應用軟體與資料庫，如 Oracle Single Instance Databases、Oracle Application Servers、Oracle Enterprise Manager 元件、協力廠商資料庫與其他應用軟體，確保了企業持續運作所必需的高可用性。

Kumar 表示，對於需要單一高可用性解決方案，保障所有應用軟體的 Linux 用戶，Oracle Clusterware 可說是完美的解決方案，其價格更是競爭對手所不及。透過 Clusterware，用戶運用一般硬體，即可達到服務所需的延展性、可用性與效能。

過去 Clusterware 只提供 RAC 用戶使用。如今用戶無須使用 RAC，即可運用 Clusterware。

甲骨文因為宣佈為 Red Hat Enterprise Linux 用戶提供支援，曾在 2006 年引發爭議，被部份人士視為在挖 Linux 散佈套件商紅帽 (Red Hat) 的牆角。Oracle Unbreakable Linux 提供企業級 Linux 支援服務，已經擁有超過 2000 名用戶。藉由為整套軟體基礎架構堆疊，與甲骨文由上至下的軟體相容性，提供單一接觸窗口，Oracle Unbreakable Linux 支援有助甲骨文提升用戶服務的回應速度，減低系統出錯時間並增加系統延展性。
