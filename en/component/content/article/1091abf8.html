Novell 表明支持 GPLv3 針對微軟立場發表聲明

Created at Friday, 06 July 2007 16:00      Written by 謝良奇

正當微軟忙著撇清自己跟新出爐的 GPLv3 沒關係之際，微軟的合作對象 Novell 卻出來表明支持 GPLv3 的立場。Novell 表示包括 SUSE Linux 企業級伺服器在內，Novell 的各項產品將納入 GPLv3 的相關技術。

 

Novell 公共關係主任 Bruce Lowry 在部落格中表示，在 GPLv3 條款下，他們重申該公司在 SUSE Linux 企業版、openSUSE 與其它 Novell 產品納入 GPLv2 或 GPLv3 授權技術，並將這些技術提供給用戶的能力。他認為，對 Novell 用戶而言，這是一項好消息，他們可以在關鍵計算上，享受到技術最為先進、最富相互操作性的 Linux 平台。Novell歡迎並支持 GPLv3，也計劃在散佈套件內納入採用 GPLv3 授權的程式碼。

GPLv3 釋出後不久，微軟隨即發表聲明，表達該公司並不受限於 GPLv3，GPLv3 也不適用於該公司的立場。微軟更表示，為避免任何可能的法律問題，該公司決定發佈的 Novell支援憑證將不授予持有者接收來自 Novell 或其它任何一方有關 GPLv3 程式碼的支援或更新。

Lowry 認為，微軟目前的立場是希望消除任何有關 GPLv3 對微軟適用性的含糊認知。不論微軟的立場為何，Novell 希望對其用戶重申該公司的決心，Novell 一定會繼續散佈具備完整功能與特色的 SUSE Linux 企業伺服器，包括採用 GPLv3 授權的各種元件。

往後透過微軟的憑證取得 Linux 的用戶，不論微軟提供的憑證條款為何，Novell 都會提供標準的 SUSE Linux 企業級伺服器訂戶服務。至於已經從微軟取得 SUSE Linux 企業版憑證的用戶，Lowry 表示因為這些憑證是在 GPLv3 釋出前就發出，將不會受到任何影響。Novell 將持續以用戶的需求為優先考量，保證用戶能在企業中應用最新的 SUSE Linux 企業版。

Lowry 表示，Novell 與微軟計劃持續雙方在虛擬技術、標準系統管理、身份確認互通性及文件格式轉換上的合作開發，有關 Novell 和微軟合作協議中免於訴訟一項在 GPLv3下的適用性，Lowry 說雙方面的用戶將可持續受到此一條款的保障。對於 Novell 的用戶，所有 Novell 的產品，不論散佈管道為何，不論是採用 GPLv2 或 GPLv3 授權，都受到微軟免於訴訟的合約保障，
