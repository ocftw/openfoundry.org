FSF 推出完全自由的 Ubuntu 衍生散佈套件 gNewSense

Created at Saturday, 04 November 2006 08:00      Last Updated on Friday, 09 March 2007 01:43 Written by 謝良奇
為了持續實踐自由軟體的理念，自由軟體基金會 (The Free Software Foundation, FSF) 宣佈釋出 Ubuntu 散佈套件的衍生版本 gNewSense 1.0，其中移除了所有私有軟體。這套由 Brian Brazil 與 Paul O'Malley 兩名愛爾蘭自由軟體提倡者創造的 Linux 散佈套件，期能提供用戶一套融合 Ubuntu 穩定性同時保障自由軟體的軟體套件。 gNewSense 開發者在釋出公告中表示，從理念角度出發，他們希望能創造提供系統上所有軟體原始碼給用戶的 GNU/Linux 散佈套件。gNewSense 專案目前由自由軟體基金會贊助。 在 gNewSense 1.0 中，私有韌體與字型全都遭到移除，也取消了對非自由 Ubuntu 軟體庫的存取。散佈套件內含全新的圖片，預設安裝也納入了開發者導向的套件如 Emacs 與 GCC 編譯器。Ubuntu 使用的私有開發管理工具 Launchpad，在 gNewSense 中也不復得見。 自由軟體基金會的自由軟體目錄維護者 Ted Teah 認為，藉由移除所有私有核心韌體、受限的軟體庫與 Ubuntu 的私有散佈套件管理工具，才能完成這套 100% 自由的 GNU/Linux 散佈套件。 儘管如此，許多 Ubuntu 支持者認為 gNewSense 的做法是多此一舉。因為 Debian 長久以來就以理想路線著稱，gNewSense 開發者只要投入 Debian 開發即可。甚至可以在 Ubuntu 安裝程式加入允許用戶是否安裝私有韌體的選項，都是更合理的做法。Ubuntu 的非自由 "Multiverse" 軟體庫也非預設安裝項目之一。此外，更有人擔心 gNewSense 拒 Launchpad 於門外的態度，讓 gNewSense 未來的修補與改善，難以併入 Ubuntu 中。 Brian Brazil 提出他的解釋，首先他指出一套完全自由的 Ubuntu 衍生散佈套件絕對有其必要。他援引 gNewSense IRC 頻道上隨著 1.0 版釋出後越來越多的用戶數，證明對於這樣一套散佈套件的需求遠超過反對者的想像。他認為 Debian 因為核心內含非自由韌體同時提供用戶非自由軟體元件的選項而非 100% 自由，在未來的 Etch 版本中更要納入私有驅動程式。Brazil 更指出即使只是在 Linux 散佈套件中包含私有軟體選項都是直接支持與鼓勵非自由軟體的做法。 gNewSense 1.0 目前已提供下載 LiveCD ISO 格式，此外，gNewSense 團隊提供用戶名為 Builder 的軟體可供建構基於 gNewSense 的客製散佈套件。 相關網址：
