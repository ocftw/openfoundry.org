VMware 宣佈購併 SpringSource

Created at Tuesday, 11 August 2009 08:00      Last Updated on Sunday, 23 August 2009 21:53 Written by 謝良奇/編譯

VMware 宣佈購併企業應用軟體與 Java 開發公司 SpringSource，此舉將協助該公司擴展其開放源碼支援，並開發內部與外部雲端運算架構。預料 VMware 在雲端運算應用軟體管理與開放源碼社群中，將變得更為舉足輕重。

此次購併使 VMware 在開放源碼社群中將趨於重要角色，並給予其新技術，以便與擁有自家虛擬化技術，並且將於近日接管 Sun Microsystems 和 Java 技術的 Oracle 進行競爭。

身為虛擬化領導廠商的 VMware 正設法轉為雲端架構作業系統大廠，VMware 表示，該公司因為 SpringSource，將跨足在虛擬化、應用程式框架和雲端運算的交叉口。

VMware 表示將支付約 3 億 6200 萬美元現金與股票，同時為 SpringSource 的購併發行約 5800 萬的股票選擇權。購併預期會在本季告一段落。VMware 表示，購併已獲得 SpringSource 股東們的同意。

SpringSource 是一家以開放源碼技術為基礎應用軟體的開發商，帶領許多開放源碼社群，其中包括著名的企業 Java 程式設計模式 Spring Framework，根據 VMware 的說法，近半數的企業 Java 專案都有來自 Spring Framework 的支援。Spring Framework 全球開發者使用人數近 2 百萬，是一套輕量級程式設計環境，讓應用軟體獲得跨越開放源碼與商業應用伺服器環境的可攜性。

此外，SpringSource 也是 Apache Tomcat Java 應用伺服器環境的關鍵貢獻者，該公司提供了該專案超過 95% 以上的程式臭蟲修補。該公司也帶領了動態語言 Groovy 與 Web 應用軟體框架 Grails。VMware 表示，富比士全球二千大企業 (Global 2000) 中，相當多企業是 SpringSource 的用戶。

購併 SpringSource 讓 VMware 在推動軟體相互運作性上，成為主要的參與者。對於正運用其虛擬化專長，讓用戶透過其 vSphere 產品轉向雲端運算的 VMware，軟體相互運作性是相當重要的一環。

vSphere 是 VMware 日前公開的新一代虛擬化技術，用戶與合作夥伴透過此一技術，將可虛擬化資料中心資源以建構雲端運算基礎架構，整合並管理處理器、儲存設備與網路，作為動態作業環境。

對於一直以來不為其軟體提供 VMware 虛擬化環境支援，而傾向讓用戶採用其 Oracle VM 虛擬化技術的 Oracle，SpringSource 給了 VMware 與 Oracle 競爭的新武器。

SpringSource 預料會完成 VMware 的平台即服務 (platform-as-a-service) 計畫，推出整合性平台，作為整併 SpringSource 軟體與 VMware vSphere 雲端作業系統的服務。

VMware 表示，該公司將持續支持 SpringSource 解決方案背後的原則，那就是 SpringSource 軟體與其他中介軟體的相互運作性，以及對於開發者社群重要的開放源碼模式。
