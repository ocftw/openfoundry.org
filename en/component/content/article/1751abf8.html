Linux 版 Firefox 出現使用者授權合約 引發 Linux 用戶不滿

Created at Thursday, 18 September 2008 08:00      Written by 謝良奇/編譯

想不到在 Linux 版本 Firefox 中，加入一個 Windows 與 Mac OS X 使用環境下常見的使用者授權合約 (end-user licensing agreement，EULA)，竟引發 Linux 用戶的高度不滿。開放源碼開發組織 Mozilla 承認其錯誤，將在該瀏覽器下次更新時，移除此一法律文件。

事件起因於 Ubuntu 用戶上週注意到新的 Firefox 版本 3.0.1，隨附了一份授權合約，並表達其不滿。Ubuntu 商業贊助商 Canonical 執行長 Mark Shuttleworth 很快就在某個郵件論壇上表示，雖然 Mozilla 視 EULA 為必要的商標保護手段，但他不認為 EULA 會是最好的執行方案。

很顯然 Ubuntu Linux 用戶的態度，是不想要在其最愛的 Linux 桌面上，看到像 Firefox EULA 這樣的東西。部份 Ubuntu 用戶甚至要求該散佈套件移除 Firefox。對此 Shuttleworth 呼籲用戶冷靜，並且與 Mozilla 共商找出各方都可接受的可行解決方案。他在郵件論壇的發言中表示，咆哮與叫嚷一點也沒有建設性，你們的軟體自由，正如 Mozilla 對於 Firefox 的商標權，是建立在法律基礎上。現在誤以為你們的權利受到侵害，等於誤解了自由軟體的意義。

前 Mozilla 執行長，現任 Mozilla 基金會 (Mozilla Foundation) 主席 Mitchell Baker，出面表示，此時最重要的，就是公開承認授權合約的內容的確是錯誤的。很清楚的，正確內容應該是，這份程式碼受自由與開放源碼軟體 (FLOSS) 授權管轄，而非目前使用的一般使用者授權合約條文。雖然他們製作了引用 FLOSS 授權的授權條文，但因為並未交付給 Ubuntu 與其他散佈商並公開尋求查核，以致於犯了大錯。

不久後，Baker 更進一步表示，他們了解到任何類似於 EULA 的文件，即使是其內容基於 FLOSS，都會造成困擾，因此他們會將其移除。

目前 Mozilla 副總裁兼法律總顧問 Harvey Anderson 已經釋出一份新的 Firefox 授權提案，以徵求社群意見。Anderson 表示，這些意見指出此一方案不夠完善，他們正在重新檢討，納入社群與部份 Linux 散佈商的建議。據悉，除了 Ubuntu 之外，紅帽 (Red Hat) 與其社群分支 Fedora，也加入了草案制定的行列。

對於 Mozilla 下的 EULA 決定，Anderson 認為其中犯了兩個錯誤。首先，該公司將 EULA 置於一群不慣於這類合約的使用者面前。其次，該授權合約與 Linux 社群許多用戶以及該公司的價值並不一致。

Anderson 與該公司仍在商討呈現方式。Anderson 指出，在軟體執行時，不會有點選畫面或授權合約出現。他們會在首次執行頁面上加入部份文字，或者設置可連接到瀏覽器靜態頁面的資訊框，以注載使用者的權利。Anderson 除了表示最後實作方案仍未底定外，也張貼幾張螢幕快照。Anderson 並且貼出了授權條文修改後的草案版本。

在此同時，Firefox 3.0.2  開發版本中發現的許多 bug，將延宕該版本的釋出時程。剛好給了 Mozilla 一個機會，在該版本釋出前，加入更改過的授權條文。
