展望 2004—各大企業將陸續採用桌面 Linux

Created at Sunday, 04 January 2004 08:00      Last Updated on Friday, 09 March 2007 00:04 Written by 謝良奇
回顧 2003 之後呢？開放源碼、Linux 未來在 2004 一年內的展望為何？

首先，將焦點擺在 Linux 身上。過去一年 Linux 在企業與桌面表現均優，Linux 成為中小企業應用選擇之一。隨著 Linux 核心 2.6 的推出，這會是 Linux 技術上的一大進展。幾乎所有企業核心任務，將可由 Linux 加以應付自如。

預料不少較大型企業將開始大量採用以 Linux 為基礎的桌上系統。採用的桌面系統包括 Sun 的 JDS、Novell/SuSE/Ximian 桌面，甚至是 GNOME、KDE 等。

桌面 Linux、遊戲、嵌入式 Linux，都是 2004 預期備受關注的領域，預料將有不錯的表現。

不少開放源碼界的著名權威，如 Eric Raymond、Samba 的共同創辦人 John Terpstra 和 Matthew Johnson 等人，對 Linux 2004 的預期進展，表達了他們的看法。Eric Raymond 認為，藉若干政府採納計畫之助，辦公室軟體將是 2004 年成長最快的 Linux 應用軟體。John Terpstra 則表示 Linux 在檔案、列印服務、目錄管理與服務及群組軟體解決方案，會有亮眼的成長，這一點將是拓展桌面領域的助力。

Matthew Johnson、Florian Cramer 認為，伺服器與企業應用，仍會是 2004 年成長最快速的領域。

Tobias Weisserth 等人也視辦公室軟體，為 2004 年成長最迅速的應用軟體之一。此外，遊戲、嵌入式 Linux、群組軟體等企業應用軟體架構等，也備受青睞。

對於在桌面系統上，Linux 是否會在 2004 年達到重要進展，Raymond 抱持肯定的看法。他認為如麻薩諸塞州、巴西、南非等政府單位的採用，正推動這一股趨勢。此外，私領域內同樣有這樣的轉變發生。由於希望保密以維持優勢的想 法，這些採納者不會選擇對外公開。同樣的，Terpstra 將政府單位的採納做為桌面 Linux 的推動指標與助力。Terpstra 預期至少將有一個美國重要政府單位，會採用 Linux 桌面系統，這將掀起帶頭作用。不少政府會採用 Linux。Terpstra 預測到了 9 月，許多應用軟體將一股腦地宣佈移植到 Linux 上。

Raymond 和 Terpstra、Marcel Leal、Matthew Johnson 等人，都認為 Suse Linux 會是 2004 年最有潛力的散佈套件之一。由於 Fedora 提供 apt for RPM 之故，Raymond 也將 Fedora 列入 2004 年預期成長表現最好的散佈套件之一。

John Terpstra 認為 OpenOffice 會成為 2004 最酷的新 Linux 應用軟體，這全賴於 OpenOffice 將發表的加強功能，允許該軟體應用於基於 DocBook 的自動化出版技術，新版 OpenOffice 將提供可為企業辦公室用戶定義全新標準的協同功能。其它人心目中的 2004 最酷軟體相當分岐，其中包括了 Firebird、Xserver、EverQuest、PostgreSQL、GNOME 2.4 等等。

針對眾所矚目的 SCO 訴訟，Raymond 認為此事不會減緩 Linux 的發展，Matthew Johnson 則表示也許在一開始的 6 個月內有影響，但長期則無。

Novell 在 IBM 支援下，以 2.1 億美元收購 SuSE Linux，宣示了該公司介入桌面 Linux 的決心和企圖。Novell 在 2004 將有許多 Linux 相關計畫。Nterprise Linux Services 的下一版本，將超越 1.0 版的命令列界面，以 GUI 作為基礎。2004 年 Novell 也計畫針對旗下的 ZenWorks 管理軟體，和由於收購 Ximian 所獲得的 Linux 管理軟體 Red Carpet，提供直接連線的能力。

SD Forum 日前舉辦的活動，開放源碼：實務問題之實用方案（Open Source: Practical Solutions for Real World Problems），邀集了許多產業界人士，討論 2004 年的開放源碼動向和展望。與會者包括 Novell Linux 事業部部長 Matt Asay、Sleepycat Software 執行長 Mike Olson，和 Red Hat 商務發展副總裁 Mike Evans 等人。

針對 2004 年開放源碼發展和軟體銷售、行銷的轉變，Mike Evans 認為開放源碼背後的全球發展模式，將對大型企業造成巨大影響。越來越多新的開發者熟悉開放源碼的開發模式，與大企業原有專案模式顯得截然不同。Evans 提到開放源碼社群重要元素，那就是高品質程式開發者可得性的全球化現象。此外，開放源碼的散佈模式，擁有傳統軟體供應商的供應鏈系統所缺乏的優勢。

Sleepycat 擁有著名開放源碼資料庫 Berkeley DB，該公司執行長 Olson 不認為服務和支援會是開放源碼的最大弱點。他說，那些熟知整套軟體結構的可靠核心人物，是開放源碼難以置信的資產。當你需要支援，總有經驗豐富的專家來幫 忙。Olson 表示，Sleepycat 會立即回覆那些深入的問題。反之，舉例而言，在 Oracle，只有對於一些簡單的問題，才能有如此的支援服務。

對於 2004 年來說，Linux 是否在桌面應用上站穩腳步，是令人關注的主要問題。各大廠的動作頻頻，如 IBM、Red Hat 和 HP 等，在去年一年裡，為桌面 Linux 的推動，做出許多重要的努力。這些公司的支援，勢必有助增加桌面 Linux 系統對企業的吸引力。Sun、Novell 相繼推出或計畫企業級桌面系統與軟體，加上 OpenOffice、Gnome 等桌面軟體、環境的成熟，也將是桌面 Linux 的助力。
