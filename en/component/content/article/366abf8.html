微軟申請 XML 檔案格式專利 自由軟體未來受挑戰

Created at Sunday, 01 February 2004 08:00      Last Updated on Friday, 09 March 2007 00:58 Written by 謝良奇
“專利”將對自由軟體未來構成威脅？Bruce Perens 對此事提出警告。當大型軟體商掌握軟體開發相關專利，未來小型軟體公司、社群、個人，恐怕無法自行發展軟體。社群網站 Slashdot.org 上，一篇標題為“微軟向紐西蘭申請 XML 文書處理檔案專利”的文章，給了未加警覺的開放源碼社群，一個值得深思的提醒。一旦該專利通過，未來除了微軟，其它人無法將 XML 檔案格式，用於文字處理檔案儲存。

此事引起各界注意，News.com、 Slashdot、LinuxWorld、PCPro 和 Grolaw 各自提出報導。News.com 表示微軟在若干地區申請專利，是為了控制其它應用軟體使用 Office 新 XML 檔案格式的可能。專家們認為，OpenOffice.org 和其它應用軟體，可能因此失去與 Office 間的相互協作性。紐西蘭政府正在為政府單位，制訂文字檔案規範書，內容包括以 XML 儲存文字處理文件，而非 Word 格式，NZOSS（紐西蘭開放源碼社群）認為，當微軟取得 XML 檔案格式做為文書處理檔案儲存的專利，希望相容於微軟軟體產生檔案的軟體，將必須取得微軟授權。這一來將與紐西蘭政府希望擺脫受控單一廠商的原意，背道而馳。同時，微軟也對歐洲專利辦事處(European Patent Office），申請性質相似的專利，專利號碼為 EP1376387。這份專利申請的重點放在微軟的 XSD 檔案。XSD 檔案是 XML Schema 檔，為解譯 XML 檔案所需。對照於微軟公開其 XSD 檔，授權書更允許“免費“使用，申請專利此事顯得更為可慮。一直以來，微軟靠著專利大賺其錢，這次為何微軟申請專利，卻只為了免費提供授權。NZOSS 提出一種可能，微軟有意使該份授權，與 GPL 互不相容。該授權書的 FAQ 寫道，某些開放源碼授權可能包含某些條件、限制，不允許在 Office 2003 XML 參照 Schema 授權下的開發作業。如此一來，GPL 應用軟體將無法讀取、寫入微軟的 XML 格式檔。GPL 正是 OpenOffice.org 的雙重授權之一。OpenOffice.org 同時也是微軟在辦公室軟體上的主要競爭者。Bruce Perens 在 BBC 某次訪談中指出，軟體專利將威脅 Linux。這是比 SCO 更嚴重的問題。
