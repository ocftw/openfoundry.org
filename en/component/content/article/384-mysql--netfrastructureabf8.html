MySQL 收購 Netfrastructure

Created at Tuesday, 21 February 2006 08:00      Last Updated on Wednesday, 09 May 2012 10:25 Written by 謝良奇
感受到 Oracle 步步進逼的威脅，MySQL 日前收購了 Netfrastructure，更重要的是藉此延攬到資料庫大師 Jim Starkey。

以創造 InterBase 資料庫與後續的 Firebird 專案聞名於資料庫圈子的 Jim Starkey，宣佈其公司 Netfrastructure 已被 MySQL 收購。預料 MySQL 將藉重 Starkey 的長才為 MySQL 發展自有的交易引擎，取代現有的 InnoDB 引擎。 Oracle 去年收購了 InnoDB，近來又收購 Sleepycat Software，一連串的收購讓 MySQL 的企業用戶擔心可能因為使用 MySQL 的 InnoDB 而必須付出額外的授權金。 Netfrastructure 是一家提供結合 web-based 應用伺服器前端與 Firebird 資料庫架構重實作的公司，這套架構是由 Jim Starkey 所設計，此外 Jim Starkey 也是 Datatrieve、Rdb/ELN 與 InterBase 的主要開發者，InterBase 於 2000 年時轉變成為開放源碼專案 Firebird。 由於 InnoDB 是 MySQL 5.0 中的核心部份之一，當 Oracle 收購 InnoDB 的消息傳出後，MySQL 顯然處於極為不利的位置。Firebird 社群便開始討論 MySQL 應該改採 Firebird 作為其企業級交易引擎。 Database Journal 撰稿人之一的 Ian Gilfillan 猜想 MySQL 給 Starkey 的第一個任務，將會是開發外界期待已久的交易引擎。 Jim Starkey 本身的說法是，在 Netfrastructure 被 MySQL 收購後，他將擔任 MySQL 的全職員工，未來他不會在 Firebird 開發扮演主動角色。 他認為 MySQL 與 Firebird 從來就不是相互的競爭對手，未來應該也不會有所改變。兩者的開放源碼哲學、技術、用戶群和賣點都不相同，唯一有所共同競爭的一點，乃是在於提供最好的支援給各自的用戶。
