■ [源碼秘技] JStock－開源的股票看盤軟體

林雪凡／文

軟體名稱：JStock
介紹版本：1.0.7
官網：http://jstock.sourceforge.net/
授權：GPL-2.0


簡介

JStock 是一款開源的股票看盤與管理軟體。

以一般市面上的股票軟體為標準，JStock 的功能介面可說是相當地簡單。不過正因為簡單，這也反而容易上手。

您可以用 JStock 管理股票的買入賣出狀況，可以用它監視一系列您有興趣的股票，可以在股價低於或高於某數值時送出警告。此外您還可以設計一些「指標」（如 CCI 買進訊號），用這些指標來一口氣過濾分析大量的股票。當然，基本的技術分析線圖也是有的。
關於 JStock，就聽我介紹一下心得吧。  


安裝與執行

JStock 支援 Windows、Mac OS X、Linux 與 Solaris。請到此處下載程式檔案：http://jstock.sourceforge.net/index.html。

如果是 Linux 版本，下載後不用安裝，直接執行 jstock.sh 就能啟用。由於本程式用 Java 撰寫，您必需要事先準備好 Java 環境。實測結果不需要 Oracle Java，Open JDK 就可以了。此外也別忘記打開 jstock.sh 的執行權限再執行。

如果是 Windows 版本，則會要您將 JStock 裝進電腦才能使用。此時若您的電腦中沒有 Java 環境，JStock 會很貼心地自動幫您下載下來安裝。不過不知為何下載安裝的版本會有亂碼，所以還是自己動手抓來裝比較乾脆。您可以從這邊（http://java.com/zh_TW/download/index.jsp）下載到最新版的 Java 環境安裝檔。

建立觀察表

「觀察表」是用來即時看盤用的。您可以有多個獨立的觀察表。

要將某支股票加入觀察表，只須要在股票輸入欄中輸入股票代號（如中鋼的代號是 2002.TW）然後按 Enter 就可以了。

代號可以在以下位置查詢：http://tw.stock.yahoo.com/h/kimosel.php

其實在 JStock 的原始設計中，本來也可以不輸入代號而直接輸入股票名，不過在台股股市中，很遺憾地，這股票名可是全英文的，不能用中文（奇妙的是陸股可以用簡中顯示）。我正在和 JStock 開發者提交問題和部份的 Patch，可是開發者可能太忙了暫時沒理我，看看這問題之後能不能解決吧。

除了基本的數字以外，如果在觀察表中的某支股票上雙擊，還能叫出歷史線圖供我們做一些進一步的技術分析。如下：

記錄投資組合

請切換到【投資組合管理】頁籤。
(http://www.openfoundry.org/images/130326/jstock/JStock_06.jpg)
▲ 圖6：投資組合管理畫面，分為買入和賣出兩部份。如果看不到下方的賣出，請把視窗拉大一點。

這裡的投資組合，其實就是一個股票專用的記帳表。當您買賣股票時可以同步在這裡登記。JStock 會自動利用目前股價計算您的營利狀況、畫出不同時間賣出時的獲利圖，也可以同步記錄手續費與稅金（需到選項視窗中打開才有這功能）。

記錄的方法為：點擊下方的【買入】、【賣出】按鈕。
(http://www.openfoundry.org/images/130326/jstock/JStock_07.jpg)
▲ 圖7：點買入按鈕後出現的視窗。注意：如果您沒有打開設定畫面中的「記錄經紀費」功能，經紀費等欄位就不會出現。

記錄好之後，JStock 會自動將您的資料與當前市面上的現值做比較，看您有多少帳面盈利（您還沒賣出的）與實際盈利（您已經賣出的）。

透過賣出按鈕可以做出賣出動作，操作方式與買入按鈕雷同。
(http://www.openfoundry.org/images/130326/jstock/JStock_08.jpg)
▲ 圖8：JStock 會自動計算出您當前的盈利狀況。

您還可以透過右鍵選單，要求 JStock 繪出營利狀況變化圖來。算是很貼心的功能了。


掃描股票指標

JStock 對短線投資者提供了一個有趣的功能：掃描股票指標。

掃描股票指標是說，您可以指定一個或多個指標條件（如 CCI 買入訊號），然後用這個條件來掃描您的觀察表或整個股市。有命中的股票就會被篩選出來。

做法很簡單：首先進入【股票指標編輯器】頁籤，點擊【安裝】按鈕，選擇【從 JStock 服務器安裝】，再從服務器下載您需要的指標。

完成後切換到【股票指標掃描儀】頁籤，按掃描，選擇想掃描的範圍。
(http://www.openfoundry.org/images/130326/jstock/JStock_09.jpg)
▲ 圖9：用 JStock 掃描整個股市尋找買入訊號。

如前所述，JStock 非常輕量，主要功能就只是這些。如果您對看盤軟體沒有更多複雜的要求，甚至早就在嫌看盤軟體太肥太慢，這就快來試試它順不順手吧！

■ [自由專欄] Microsoft Windows 8 UEFI Secure Boot 申訴的正反面意見

謝良奇／編譯

日前代表 8 千名 Linux 用戶的西班牙團體 Hispalinux，向歐盟委員會提出申訴，指稱 Windows 8 硬體所需的 UEFI Secure Boot 為阻礙機制，限制了消費者在作業系統上的選擇。Hispalinux 發言人 Paul Brown 表示，身為 Linux 用戶與專業人士，他們發現要在設有 Microsoft UEFI Secure Boot 金鑰的電腦上，以 Linux 或其他作業系統開機，是件相當麻煩的事。而市場上 90% 的電腦都有此設定。

Brown 指出，不管是個人、開發者團體或公司，都必須向 Microsoft 要求許可，並等待其答覆，更別提所獲得的許可，未來有可能隨時單方面被 Microsoft 撤銷的恐懼。

波士頓大學法學院法學教授、反壟斷專家 Keith Hylton 表示，根據歐盟法律，這項投訴很可能會成立。因為在該地區，擁有領導地位的公司，很可能有責任支持競爭對手的產品。Hylton 說，在美國，法律上領導企業沒有責任對競爭對手的產品提供支援，但是在歐盟情況並不是那麼明確，因此在歐盟法律下，這樣的主張或許有部分合理性。

有關 Microsoft 市場主導地位的另一件案子，是針對該公司未能遵守提供歐洲用戶瀏覽器選項，遭歐盟重罰高達 7 億 3100 萬美元的罰款。歐盟在 2009 年對 Microsoft 提出該要求，為的是解決有關該公司在 Windows 桌面作業系統上，搭售 Internet Explorer 的競爭疑慮。

不過，在 UEFI 一事上，歐盟競爭委員會委員 Joaquín Almunia 在 1 月時提到，他並未發現有證據顯示，Microsoft 的安全需求在實際上違反了歐盟的競爭法。由於實際性、法律、經濟層面的考量都必須納入考慮，Almunia 表示原始設備製造商 (OEMs) 顯然可以提供關閉 UEFI Secure Boot 的選項給用戶。

在資訊安全廠商 Sophos 擔任顧問的 Linux 用戶 Paul Ducklin 表示，用戶雖然可以載入不同的作業系統，但過程並不簡單，特別是不熟悉技術的用戶。你可以關閉 Secure Boot，好讓你載入任何軟體系統，你也可以上載自己的 Platform Key，自行管理設備的安全性。

Brown 認為，無論如何應該有其他替代方式。該項功能應該預設為關閉，假如用戶有需要，可以按照指示開啟使用。因為要關閉 Secure Boot 對非技術用戶而言並不容易。

根據 Hylton 的意見，雖然 OEM 廠商提供用戶關閉 UEFI 選項這一點，或許會降低歐盟針對此一申訴立即行動的可能性，但並不會使 Hispalinux 的法律論據失去效用。他說，歐盟委員會的意見指出，至今沒有事實依據支持針對 Microsoft 的控訴，但這並不表示原告的理論缺乏法律基礎。

Hispalinux 的 Brown 表示，該申訴涉及 Microsoft 運用其影響力，左右製造商納入附加專屬 Microsoft 金鑰的 UEFI Secure Boot。此一機制，在實質效果上，妨礙或嚴重阻礙了在沒有 Microsoft 明確許可時，以其他作業系統進行開機的動作。

而為了達到此一目的，Microsoft 運用了它在市場上的所有影響力，迫使電腦與零件製造商接受該公司在 UEFI Secure Boot 金鑰產生系統的壟斷。

根據 Hispalinux，這項功能將對公民所有技術自主化的機會造成損害，將其角色弱化為僅僅作為被動用戶，並且使該機器成為僅有單一用途的電器設備。Brown 指出，此舉也損害自由競爭，弱化技術產業，為歐洲帶來更多的貧窮與失業問題。

Microsoft 在聲明中指出，UEFI 是用來改善電腦安全的產業標準，此一方式已經公開了一段時間。他們樂於回答任何額外問題，且他們有信心，該公司的做法符合法律規範，並有助保障客戶安全。

■ [自由專欄] 何謂開放源碼的雲端？

謝良奇／編譯

談起雲端，此一術語被誤解的程度，或許會令許多產業人士感到訝異，它通常被認為只是流行語或是毫無內涵的名詞。儘管遭到濫用，雲端事實上確是一個重要概念，而且肯定不會是曇花一現的風潮。

美國國家標準與技術研究所 (National Institute of Standards and Technology，NIST) 對於雲端運算給了相當不錯的定義，其中可分為五大特點：

＊依需自助服務 (On-demand self-service) - 意味著用戶無需和他人進行互動，即可自行配置服務。用戶提供信用卡給 Amazon、Dropbox、Contegix 等雲端供應商後，就可立即使用這些服務。

＊廣泛的網路存取 - 雲端提供的功能遍及整個網路。

＊資源共用 - 眾多用戶共用運算、儲存、網路等資源。用戶不需要知道資源配置的細節。

＊快速彈性 - 資源可依照需求快速擴增或縮減。對終端用戶而言，資源看似無限或趨於無限。

＊計量服務 - 用戶可以查詢他們用了多少資源，並且依此付費。

另外，廣泛的網路存取中部分涉及到了 API 的概念。假如服務未公開 API，就不算真的雲端。雲端服務，特別是開放雲端服務，應該能夠提供用戶程式化存取。


雲端服務有幾種常見的類型：

軟體即服務 (Software-as-a-Service，PaaS)：像是 Dropbox、Google Docs、Salesforce.com、ownCloud 都算是 SaaS。它們是提供網路為基礎應用程式或服務的軟體。在此服務下，相當多細節從用戶端被抽象化，用戶無需知道背後的作業系統為何，或耗用了多少系統資源，用戶也不必擔心軟體升級與軟體相依性等。

平台即服務 (Platform-as-a-Service，PaaS)：PaaS 是替開發者照料基礎架構、中介軟體、系統流程的服務或堆疊，好讓他們專心在應用程式開發上。例如 Google AppEngine、Engine Yard，或是開放源碼的 OpenShift。

架構即服務 (Infrastructure-as-a-Service，IaaS)：最後則是 IaaS。用戶可以撥用運算、儲存、網路資源，但底層細節仍然被抽象化。舉例來說，你可以使用 CloudStack 或 Amazon EC2 配置一個相當於 2 顆 2.0 GHz 的 Xeon CPU，4GB 記憶體，100GB 儲存空間，加上一個公開 IP 位址。

在開放雲端陣營，著名的 IaaS 包括 Apache CloudStack、Eucalyptus、OpenStack。非開放陣營則有 Amazon Web Services EC2 與 Google Compute。

IaaS 的好處在於，IaaS 用戶不需要知道它的工作原理，可是如果你想要自行部署 IaaS，了解其做法會很有幫助。最簡單的方式是把 IaaS 雲端想成一種後設作業系統，它會指揮個別 hypervisor、網路元件、儲存裝置或伺服器的運作，就像是 Linux 負責管理你的伺服器、桌機、筆電、行動裝置一樣。

聽起來雖然複雜，而且設定 IaaS 的確並不簡單，但並非你所想像的那麼高難度。拿 Apache CloudStack 當例子好了，會有一套應用程式在一台或多台主伺服器上運作，與 hypervisor、儲存設備、網路裝置溝通。它提供了 API 或基於 Web 的使用介面，供管理者與用戶管理資源。

對於大規模操作或必須程式化管理資源，以及需要讓用戶自行依需配撥資源，並隔離個別用戶資源的環境，甚至為了避免濫發管理權限，這種做法都具有相當強大的功能。因為 Linux 與開放系統才使得此一規模成為可能，且對於運行今日的組織與應用程式來說，都是不可或缺的。

擁有開放的雲端之所以事關重大，在於我們需要在更大規模上，延續 GNU、Linux 社群過去 20 多年來的努力成果，也因為我們必須讓這朵雲比 Amazon 或其他私有公司更加龐大，更因為用戶與組織應該在大規模運算上，享有和過去在單一伺服器時同樣多的控制權。

所以說，雖然許多人或許已經厭煩再聽到雲端這個雲端那個了，但雲端不會這麼快結束。如果你關注軟體自由的話，這就是新一代的軟體自由。

■ [源碼新聞] IBM 雲端產品轉以 OpenStack 為基礎

謝良奇／編譯

IBM 宣布將以開放源碼為基礎，特別是 OpenStack，重新架構該公司旗下的雲端服務與軟體。IBM 資深軟體副總裁 Robert LeBlanc 表示，和被徹底顛覆的 Web 與 Linux 一樣，開放標準與開放源碼同樣會為雲端運算帶來巨大的影響。

IBM 此舉被許多人拿來和該公司在 Linux 初期即採納該系統相提並論，然而和 Linux 不同的是，如今整個產業已經普遍採納開放源碼或者開放標準雲端。如果 IBM 能夠更為有效的推動標準與開發，那麼在其雲端產品上致力投入運用開放源碼基礎，對 IBM 才有意義。

作為該公司此流程的第一部分，IBM 推出了以 OpenStack 為基礎的新的私有雲端產品，IBM SmartCloud Orchestrator，其設計透過雲端交付模式與圖形化介面，得以快速的建構與部署新的雲端服務。

OpenStack 是用來管理私有或公開基礎建設服務 (Infrastructure-as-Service，IaaS) 雲的開放源碼平台。IaaS 雲以可輕鬆部署的虛擬機器和分散式儲存服務的形式，提供運算資源。最初由 NASA 和 Rackspace 所開發，OpenStack 被發展為一項開放源碼專案，接著有 IBM、Red Hat，甚至是 VMware 等公司的加入而成為蓬勃社群。由於來自於其周圍社群的壓力，OpenStack 之前被移交給獨立基金會負責管理其發展。OpenStack 最近一次釋出是去年 9 月的 Folsom 2012.2。目前處於開發階段的下一次釋出，則是預定於 4 月的 Grizzly。

■ [源碼新聞] Python Software Foundation 透過協商守住歐洲的商標 

黃郁文／編譯

The Python Software Foundation (PSF) 日前宣布，與英國公司 PO Box Hosting（其股票上市名稱為 Veber）就 Python 商標爭議一事已達成和解協商！在此之前 Veber 於 1997 年租用了 python.co.uk 網域名稱，並直接以 Python 字樣來稱呼其提供的雲端服務，同時於近期申請 Python 商標在歐洲的註冊，此一舉措讓已使用 Python 字樣為其程式語言名稱的 PSF 大為震驚，從而自今年 2 月份開始，向四面八方尋求協助，以對抗 Veber 此一商標註冊行動。而令人欣慰的是，經過眾多自由開源軟體開發社群與商用公司的表態支援後，該爭議已於日前圓滿解決。

PSF 就 Python 這個字樣在資訊與軟體開發領域下的使用，已經在美國取得商標權 (trademark) 的註冊，不過過往因為經費有限，而未一併申請在歐洲的商標註冊，從而 Veber 才仍能在歐洲提出同一字樣的商標申請，依商標基於國際協議與各國法制的互惠原則來看，PSF 仍有機會主張已使用 Python 字樣多年，且於資訊領域已獲得廣大使用群眾的認同，而可以援引其具有著名商標、標章 (well-known trademark or mark) 的地位，以事後撤銷 Veber 在歐洲申請的 Python 商標，不過透過正式司法訴訟或是異議程序總是曠日費時，故在 PSF 公開徵詢各界的支援與協助後，Veber 也重新檢討了此一商標申請舉措的合宜性，雖然部份網路支持者失序發表了對 Veber 公司成員的人身威脅或是無理謾罵，但可幸最後雙方可以透過協商的方式來理性對談！經協議之後，Veber 同意改以新的品牌名稱行銷其雲端伺服器與備份服務，並同意撤回商標申請案，中止其原已運作的「http://python.co.uk」網址，並支持 PSF 在歐洲繼續使用 Python 一詞來推廣 Python 程式語言的相關專案。PSF 的主席 Van Lindberg 表示：「PSF 建置的最主要的目的，就是要對全球的 Python 開發社群提供支援，而他很欣慰 Veber 願意承認 PSF 使用 Python 字樣的合法與合宜地位，及同意迅速完成和解，並期待 Veber 產品在市場上也能有亮眼表現！」Van Lindberg 同時對熱情參與此一活動以促成和解協商的廣大 Python 社群表示感激，感謝其等提供金錢與行動上的支援，以協助爭議能儘速落幕。

而雖然 PSF 使用 Python 字樣為其標章，在資訊軟體領域已甚為著名，但為免往後再發生類似的爭端，Python Software Foundation 已透過正式流程進行歐洲商標的註冊。

■ [源碼新聞] 4 月 20 日是 2013 年的硬體自由日

謝良奇／編譯

硬體自由日 (Hardware Freedom Day) 的主辦單位已敲定 4 月 20 日為該活動第二屆的日期。由非營利組織數位自由基金會 (Digital Freedom Foundation) 於 2012 年所發起的硬體自由日，是為了建立軟體自由日 (Software Freedom Day) 的硬體版本。軟體自由日這個自由軟體的慶祝活動，同樣是由數位自由基金會主辦。該組織已經在世界各地舉辦過許多活動，讓人們了解並參與那些在他們周圍發展的自由軟體、開放源碼與社群。硬體自由日是為了促進開放硬體 (Open Hardware) 的理念，鼓勵人們不加限制地分享設計，並基於這些設計創造實際硬體。

硬體自由日正在徵求推廣活動的支持者、贊助活動的贊助者，以及在全世界各地共襄盛舉舉辦活動的發起人，當然還有樂意參與活動的人。全球各地已經有不少活動註冊參與。

例如，在英國名為 Chiphack 的活動打算在倫敦於為期兩天的週末舉行。主辦單位會為入門者開設現場可編程輯閘陣列 (Field Programmable Gate Arrays，FPGA) 的程式設計課程，同時經由從 LED 計數器開始到創造 UART 發送器的實際操作，為他們介紹硬體描述語言 (hardware description languages) 和 FPGA 工作流程。參與活動者需要攜帶 FPGA 開發板與筆電。活動贊助者 Embecosm 將贈送 DE0-Nano FPGA 開發板，給前 5 位註冊該活動的學校教師。現場也備有數量有限的開發板，供與會者使用。

距離今年的硬體自由日還有一段時間，有意願的主辦者還有不少時間可以準備不同技術水準的活動。硬體自由日主辦單位也為這些主辦者提供了入門指南。

歡迎各界對於自由／開源軟體有興趣以及熱忱的朋友們報名參加！

詳細資訊如下：


活動目標

＊以推廣自由軟體為目標，藉由此活動的舉辦，讓自由軟體的好處更廣為人知，並進而引起社群間討論、使用及研究自由軟體的風氣。

甄選辦法

■ [源碼新聞] 歐巴馬要讓政府出資的研究結果公開發表

謝良奇／編譯

白宮在一份釋出的備忘錄中表示，由政府出資的研究，其結果必須在首次釋出 12 個月後，公開發表在科學期刊上。這項美國總統歐巴馬發表的倡議還包括了數位形式的附帶科學資料，前提是其發表不影響第三方的權利，如隱私權。

該倡議僅限於由美國主要政府機構，以至少每年 1 億美金預算出資贊助的科學研究。這些機構包括美國國家衛生研究院 (National Institutes of Health，NIH) 與國家科學基金會 (National Science Foundation，NSF)。所有符合該條件的機構必須在 6 個月內，向美國科學與技術政策辦公室 (Office of Science and Technology Policy，OSTP)，針對此一開放存取 (Open Access) 倡議，提出發展藍圖與實施策略草案。一旦 OSTP 審閱過這些草案，該機構將會在定案前，徵求有關方面的意見，例如大學、圖書館、出版社。

藉由此一倡議，華府希望確保實際的科學研究資料，能盡可能地在無障礙情況下公開給大眾、產業、研究社群。白宮方面表示，此舉可鼓勵創新突破並推動經濟發展。

美國政府也承認，出版者所提供的寶貴服務，對於確保高品質科學出版品來說是不可或缺的，例如協調同儕評審的貢獻。該備忘錄指出，繼續提供這些服務是相當重要的。在此同時，美國國會正在審查開放存取法案草案。該法案規定研究資料必須在首次發表 6 個月後公開，但原始科學資料並不在此限。此一缺漏也受到美國民權運動人士的批評。

■ [源碼新聞] 微軟開源 Kinect 挽留開發者

Kenneth Mak／文

◎本文原載 Linux Pilot，原文章按此 (http://www.linuxpilot.com/industry/news/201303130101Kinect-OpenNI-SensorKinect)。

Kinect 作為新一代的體感設備，為了吸引開發者，微軟早在 2011 年就已經發布了官方 SDK，這次微軟更進一步，宣布將根據開源授權，開放 Kinect 的部分程式碼，意圖挽留正在流失的開發者。

(http://www.openfoundry.org/images/130326/Kinect.jpg)

微軟自 Windows 8 起加入了觸控功能，下一步計畫便是讓 Kinect 體感控製器，成為 PC 上的標準用戶界面設備。微軟在推出 Kinect 的當初，只想到利用它來控制 Xbox 360，但開源社區的目光更長遠，一開始已經看到 Kinect 將會是下一代的裝置輸入標準，因此就開發出開源的相關項目，包括 OpenKinect (http://www.linuxpilot.com/software/kiji/2010120301) 和 OpenNI SDK，並開發了開源的 Kinect 驅動程式 SensorKinect，自此以後 Kinect 便可以在 Xbox 360 以外的平台下使用了。

微軟眼見形勢有變，於是後來也推出了官方 SDK 。但微軟的 Kinect SDK 只能在 Windows 7 或以上的平台使用，OpenNI SDK 卻可以在 Linux 、 Mac OS X 和 Windows 上使用，硬體架構更支援 x86 和 ARM 平台，令 OpenNI SDK 成為了目前被最廣泛使用和認可的開源 Kinect 函式庫。OpenNI SDK 還有一個微軟了官方 SDK 也做不到的功能，就是支援 Kinect 以外的其他體感裝置，例如華碩的 Xtion Pro 和 Xtion Pro Live ，從一開始就是以針對 OpenNI 而開發的。

微軟深知長此下去，它將會喪失對 Kinect 的主導權，這不但是面子問題，也是利益的問題，為挽留開發者，微軟終於再度出招，2013 年 3 月初微軟 Kinect for Windows 開發者社區經理 Ben Lower 宣布中稱，通過 Apache 2.0 開源協議提供部分程式碼，包括用於面部追蹤、骨骼查看器和連續手勢的程式碼。這些程式碼由 C# 、 C++ 和 VB 編寫。微軟表示這樣做是為了讓開發者有機會重用目前的 Kinect 程式碼，並幫助微軟研究如何改進它們。此外微軟還建立了一個 Kinect for Windows 開發者社區，為有興趣開發 PC 平台體感和語音控制應用的開發者提供技術支援。不論OpenNI 今後發展如何，它的出現至少令微軟的策略有所改變，這正是開源的力量。
