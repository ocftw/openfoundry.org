reStructuredText簡易入門

應用Docutils的標記語法和分析元件

reStructuredText是個不受平台限制、任何文字編輯器都可編寫、而且容易閱讀的純文字標記語法和分析系統。reStructuredText利用Python中的Docutils元件將純文字轉換成各式檔案，再利用性高。

reStructuredText（reST）是Python文件的標準語法，適合用來編寫簡單的HTML、文件和程式中的文稿。你只要在一份純文字文件中，加上簡單的標記，再利用Python中的Docutils處理，即可得到多種格式的文件，包括：HTML、PDF、OpenOffice 格式和LaTex格式。

reST的學習速度非常快速，你只要懂得數個常用的標記語法即可，每個人都可在十分鐘內開始編寫reST文件，即使你完全不懂得如何使用Python也沒關係，因為我們只有在編寫完成，開始轉換文件時，才需用到Python指令，其他時間則可以使用任何純文字編輯器來撰寫reST文件。

reST的應用範圍非常廣泛，你可以用它撰寫簡單的HTML文件、報告、日記甚至是電子書，只要是文稿都難不倒它，不過由於reST專注於文稿撰寫，排版能力不強，但這可以用reST靈活的檔案轉換特性，先將reST轉換成LaTeX後，再進行後續排版來解決，因為你想拿它來寫書也不是完全做不到。

雖然reST的學習簡單而且應用容易，不過它的功能可不含糊，不論是Header、Footer、插入圖片、圖說或是Foot Note、Citation等都完全支援，如果熟悉Python也能夠做進一步的編寫和轉檔設定。

不過reST也不是完全沒有缺點，一是用來撰寫亟需分頁的文件會有問題，因為reST並沒有分頁的功能；二是目前reST並不支援LaTeX中的數學式，雖然一般人很少會需要這個功能，但還是會讓心中有點小小的缺憾，畢竟它的對手AsciiDoc（另一個標記語言）做得到。

除此之外，reST還可以和Zwiki搭配，應用在Zope或是Plone中，提供易用的內容編寫介面，讓你不需任何進階功能，也可以編寫複雜的內容。而許多Blog程式也提供reST格式的支援，如：Wordpress和Movable Type等，都可以利用外掛支援reST格式，讓Blog編寫更為方便、簡易。

常用reST語法

之前已經說過reST文稿的撰寫非常簡單，我們就先來看看有哪些標記是撰寫reST文件時，一定會用到的標記。

標題標記

在reST中的「標題」標記非常簡單，只要在標題（上）下使用非字母和數字的七位元ASCII記錄畫上底線（或上標線）即可，你不需擔心標題要使用何種格式，只要在相同位階的標題上使用相同的標記即可。可用的字符有：! " # $ % & ' ( ) * + , - . / : ; < = > ? @ [ ] ^ _ ` { | } ~。

例如：

========================
標題
========================
小標題
------------------------

會顯示成：

標題

小標題

段落標記

在reST中，段落只要在中間隔一行即可，Docutils會自動判別各個段落，例如：

這是一段。

這是另一段。

會顯示成：

這是一段。

這是另一段。

字體格式標記

如果想在reST中使用粗體，只要在字詞前後加上兩個*：

**粗體**

會顯示成：

粗體

如果要使用斜體，只要在字詞前後加上一個*

*斜體*

會顯示成：

斜體

逸位字元\

中文reST中不可或缺的好幫手

由於reST非常重視英文中的空格判斷，因此有時你在中文裡面直接使用reST標記無法產生效用，這是由於中文不像英語字間有空格，因此我們必須加個空格讓reST標記得以作用，這時我們就可以在空格前加上一個逸位字元\，將這個空格隱藏起來，經過Docutils處理後，你的文稿上就不會出現一個個空格了。逸位字元還可用於取消下一個標記的功能，讓它被Docutils判別為文字而非標記。

字面文字標記（inline literal）

字面文字標記`` ``可以讓文字以原有的形態出現，而非經過Docutils處理後的文字，通常以等寬字體出現。

例如：

``reStructuredText is great!``

會顯示成

reStructuredText is great!

而非

reStructuredText is great!

文字解譯標記（interpreted text）

除了粗、斜體之外，你還可以利用文字解譯標記` `為字體加上更多的變化。` `可以加上各種「role」讓字體呈現不同的變化，包含連結、上標、下標

例如：

  1. 在文字解譯標記後加上一個底線會形成連結。這時只要指定連結目標就可以形成超連結：

    `自由軟體鑄造場`_

    .. _自由軟體鑄造場: http://www.openfoundry.org/

    會顯示成

    自由軟體鑄造場

  2. 在文字解譯標記前加上:sub:和:sup:標記，可形成下標和上標字：

    :sub:`下標字`正常字

    會顯示成

    下標字正常字

    2:sup:`12`

    會顯示成

    212

  3. 在文字解譯標記前加上一個底線，可以讓標記中的文字形成一個超連結目標。

    例如：

    _`政治大學`

    會顯示為

    政治大學

    表面上看起來完全沒變，但只要你指定一個名為「政治大學」的超連結，就可以將該超連結的目的地指向為上方的文字解譯標記，這在同一文件中作交叉查詢時非常好用。

    例如：

    利用文字解譯標記建立一個名為「政治大學」的超連結如下：

    `政治大學`_

    會顯示成

    政治大學

    這個連結的目的就是上方使用文字解譯標記形成的超連結目的。

超連結

之前我們已經看到幾個使用超連結的方式，現在我們再詳加研究一下超連結的使用方式。

  1. 第一種方式非常簡單，只要直接打上網址即可。Docutils會自動幫你把網址轉成指向該網址的超連結，例如：

    打上：

    http://www.google.com.tw

    會顯示成

    http://www.google.com.tw

  2. 第二種方式是為文字指定一個超連結目標，這時必須使用上面說到的文字解譯標記，首先我們將要形成超連結的文字利用文字解譯標記包起來，並在其後加上一個底線，然後再為它指定一個目的網址，指定目的網址的語法為「..+ 空格（space）+底線（_）+超連結文字+冒號+空格+網址」例如：

    `估狗大神`_

    .. _估狗大神: http://www.google.com.tw

    會顯示成

    估狗大神

    Note

    這個用法應用在中文上有時會遇到一些問題，就是當你想連結的文字是處在一段文字之中時，由於reStructuredText必須依賴英文間的空格來判斷標記的作用點，如果你直接使用此用法會造成判斷失誤，我們可以用以下方式解決：利用逸位字元\，逸位字元除了可取消下一個標記的作用外，也可以用來取消空格，因此，將上述語法更改為「\ `估狗大神`_\ 」（逸位字元+空格+原超連結語法+逸位字元+空格）即可。

  3. 除了上方兩種方式外，我們還可以使用內嵌網址超連結，語法為「`超連結文字+空格+<網址>+`+_」，例如：

`不離不棄 被當白七 <http://blog.xdite.net/?p=228>`_

會顯示為

不離不棄 被當白七

  1. 如果你要製作成超連結的文字很長，我們也可以使用無名稱超連結，這可以省去輸入超連結名稱的時間，標記格式為「`+超連結文字+`+兩個底線」，例如：

    Python is `one of the best scripting language`__ in the world.

    .. __: http://www.python.org

    會顯示為

    Python is one of the best scripting language in the world.

    無名稱超連結也可以寫成下列更簡單的形式，適合喜歡偷吃步的人使用：

    `Apple Mac OS X is the most beautiful Operation System`__ in the Universe.

    __ http://www.apple.com

    這會顯示為

    Apple Mac OS X is the most beautiful Operation System in the Universe.

  2. 以上是外部網址的連結語法，現在我們來看看內部文件的超連結語法，首先我們必須知道所有的標題都可以當成超連結目標，因此我們只要將超連結文字取為和標題一樣即可快速製作內部文件的交互參照超連結，例如：

    `字面文字標記（inline literal）`_

    會顯示為

    字面文字標記（inline literal）

  3. 第二種交互參照的語法，只要利用指定超連結目標的語法指定一個字詞為超連結目標後，就可以利用超連結語法指向到設定好的超連結目標，請參照文字解譯標記.

  4. 間接超連結，間接超連結可以指向與其他超連結相同的網站，如此一下就不需重新輸入，例如：

    `Google`_

    .. _Google: 估狗大神_

會顯示為

Google

  1. 此外，利用無內容的超連結語法，我們也可以達成交互參照的功能，例如：

    我們先在文章首段中插入一個空的超連結目標語法如下：

    .. _回頁首:

    然後再利用超連結語法「`+超連結文字+`+_」建立一個超連結，那麼只要按一下這個超連結，就可以回到剛剛插入目標語法的下一段。

    回頁首

插入圖片

插入圖片可分為直接插入圖片和插入帶有圖說的圖片，分別解說如下：

  1. 插入圖片：我們可以利用「.. image:: 圖片名稱」這個語法來插入圖片，而且還可以設定文字對齊方式、大小、縮放百分比和超連結等。例如：
.. image:: http://lh3.google.com/Mischa.crave/RJDrBmWoABI/AAAAAAAAAAs/iXHMzjR7_1Y/s800/14195714_2816efe47d_o.jpg
   :scale: 100

會顯示為

.. image:: http://lh3.google.com/Mischa.crave/RJDrBmWoABI/AAAAAAAAAAs/iXHMzjR7_1Y/s800/14195714_2816efe47d_o.jpg
   :scale: 100
   :height: 120
   :width: 160
   :align: left
   :alt: I'm a cat.
   :target: http://picasaweb.google.com/Mischa.crave/MyCat

會顯示為

「.. image::」這個標記的參數有「:align:」（對齊）、「:height:」（長）、「:width:」（寬）、「:scale:」（縮放百分比）、「:alt:」（代替文字）和「:target:」（超連結目標）等。

  • 「:align:」可以指定圖片在文字中的對齊方式，可用的值有「top」、「middle」、「bottom」、「left」、「center」和「right」。
  • 「:height:」可以指定圖片的長，值必須為整數。
  • 「:width:」可以指定圖片的寬，值必須為整數。
  • 「:scale:」可以指定圖片縮放的百分比，值必須為整數，但後面不需加百分比符號[1]。
  • 「:alt:」可以指定圖片無法載入時的代替文字，值必須為文字。
  • 「:target:」可以指定圖片的超連結目標，值必須為一個網址。
[1]此功能必須安裝 PIL（Python Image Library）才看得出效果。
  1. 我們還可以利用「.. figure::」這個指令插入帶有圖說的圖片，這個標記的用法和「.. image::」相似，參可設定的參數也大致相同。例如：

   :width: 400
   :figwidth: 400

這是我家的肥笨貓，名叫 Lev Tolstoy ，愛吃又愛睡。我喜歡在他睡覺時拿檯燈猛照他，讓他只能遮著眼睛睡。

  1. 而如果需要在字間插入圖片，則可以使用下列方式：
因此，除了\ |b1|\ 之外，不要按下任何按鈕。

.. |b1| image:: http://tinyurl.com/2z9wts

表格的用法

在reST中，表格的繪製十分直覺、好用。你只要按照想要的格式，直接繪製表格即可，例如：

+------------+------------+-----------+
| Header 1   | Header 2   | Header 3  |
+============+============+===========+
| body row 1 | column 2   | column 3  |
+------------+------------+-----------+
| body row 2 | Cells may span columns.|
+------------+------------+-----------+
| body row 3 | Cells may  | - Cells   |
+------------+ span rows. | - contain |
| body row 4 |            | - blocks. |
+------------+------------+-----------+

會顯示為：

Header 1 Header 2 Header 3
body row 1 column 2 column 3
body row 2 Cells may span columns.
body row 3 Cells may span rows.
  • Cells
  • contain
  • blocks.
body row 4

此外，如果表格的格式較為簡單，我們還可以利用另外一種更簡易的方式編排表格如下：

=====  =====  ======
   Inputs     Output
------------  ------
  A      B    A or B
=====  =====  ======
False  False  False
True   False  True
False  True   True
True   True   True
=====  =====  ======

這種表格會顯示成：

Inputs Output
A B A or B
False False False
True False True
False True True
True True True

註解和引用

在撰寫學術文章時常用到的註腳（footnote）和引用（citation）在reST中也可以輕易地編寫出來，其reST語法如下：

  1. 註腳:

    Linkin Park [#]_ is a rock band from Agoura Hills, California. They
formed in 1996, and have since sold more than forty million albums and
won two Grammys. They first achieved mainstream success in 2000 with
their debut album, Hybrid Theory, which has since been certified Diamond
by RIAA [#]_. Their following studio album, Meteora, continued the
band's success, topping the Billboard 200's album charts in 2003, and
was followed by extensive touring and charity work around the world.

.. [#] 鈕饅頭 sucks。
.. [#] RIAA also sucks.

    這會顯示為：

    Linkin Park [2] is a rock band from Agoura Hills, California. They formed in 1996, and have since sold more than forty million albums and won two Grammys. They first achieved mainstream success in 2000 with their debut album, Hybrid Theory, which has since been certified Diamond by RIAA [3]. Their following studio album, Meteora, continued the band's success, topping the Billboard 200's album charts in 2003, and was followed by extensive touring and charity work around the world.

    [2]

    鈕饅頭 sucks。
    [3]

    RIAA also sucks.（用#標示的註解，會按照順序排下來，但是按照全篇的順序，而非上一個數字。你也可以自行在中括號後標示註腳的數字。）

  2. 引用: 引用和註腳的reST語法其實是一樣的，只是引用使用文字來代替中括號中的數字，例如:

    這是常用在期刊中的引用格式 [CIT2002]_

.. [CIT2002] 引用通常會置於文末。

    會顯示為：

    這是常用在期刊中的引用格式 [CIT2002]

    [CIT2002]

    引用通常會置於文末。

以上所介紹的只是reST其中的一小部份語法，其他更多語法可至reST官方網站中參閱，網址為：http://docutils.sourceforge.net/rst.html 。
